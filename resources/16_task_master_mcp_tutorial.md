# Task Master MCP 튜토리얼

## 목적
- AI 기반 작업 관리 도구 Task Master의 효과적 활용
- 연구 프로젝트의 체계적 관리와 추적
- AI 협업 파트너로서의 Task Master 활용법

## MCP 설치 및 설정
1. **MCP 설치**: `part2/15_mcp_installation_guide.md` 참조
2. **Task Master 설정**: MCP 서버에 Task Master 추가
3. **연동 확인**: VS Code에서 Task Master 명령어 테스트

## 기본 사용법

### 프로젝트 초기화
```markdown
새 프로젝트를 시작합니다.

프로젝트명: [프로젝트명]
기간: [예상 기간]
현재 상태: [진행 상황]
주요 정보: [핵심 내용]

다음 작업을 제안해주세요.
```

### 작업 분해
- 큰 목표를 검증 가능한 하위 과제로 세분화
- 각 작업에 예상 소요 시간과 우선순위 지정
- 의존성 관계 명시 및 선행 조건 확인

### 진행 상황 추적
- 작업 완료 시 진행률 업데이트
- 문제 발생 시 AI와 공유하여 해결 방안 모색
- 주간/월간 정기 점검 및 계획 조정

## 연구 워크플로우 적용

### 1. 문헌 리뷰 프로젝트
- **초기 설정**: 리뷰할 논문 수, 기간, 방법론 정의
- **작업 분해**: 논문별 검색, 검토, 요약, 통합 단계
- **품질 관리**: 체크리스트 기반 검토 시스템

### 2. 논문 작성 프로젝트
- **전체 구조**: 서론, 방법론, 결과, 논의, 결론
- **단계별 진행**: 각 장별 작성 및 수정 계획
- **마일스톤 관리**: 중간 발표, 제출 기한 등

### 3. 연구 데이터 분석
- **데이터 전처리**: 수집, 정제, 검증 단계
- **분석 계획**: 통계 분석, 시각화, 해석 단계
- **결과 정리**: 보고서 작성 및 검증 단계

## 실전 예시

### 시나리오: 석사 논문 연구
- **프로젝트**: 온라인 학습 참여도 분석 (6개월)
- **주요 마일스톤**: 
  - M1: 문헌 고찰 완료 (4월 말)
  - M2: 데이터 수집 완료 (5월 말)  
  - M3: 중간 발표 (6월 말)
  - M4: 논문 제출 (8월 말)

### Task Master 활용 결과
- AI가 프로젝트 상황을 분석하여 다음 우선순위 작업 제시
- 각 작업을 주간 단위로 세분화하여 관리 용이성 확보
- 진행 상황 업데이트를 통해 일정 지연 사전预警 및 대응

## AI 협업 전략

### 효과적인 프롬프트 작성
- **명확한 컨텍스트**: 프로젝트 상황, 목표, 제약사항 상세 제공
- **구체적 요청**: 원하는 출력 형식과 기준 명시
- **점진적 접근**: 큰 요청을 작은 단위로 분해하여 처리

### 문제 해결 협업
- **정확한 진단**: 문제 상황을 객관적으로 분석
- **여러 방안 제시**: AI와 다양한 해결책 모색
- **실행 계획 수립**: 즉시 적용 가능한 단계별 계획
