**타입**: 콘텐츠 슬라이드
**소요 시간**: 2분

### 콘텐츠
**제목**: "원칙 1: 명확성과 구체성"

**핵심 개념** (18pt):
- **문제**: 모호한 요청은 모호한 결과를 낳는다
- **해결**: 5W1H 원칙 적용
  - Who (누가): 대상 독자/사용자
  - What (무엇을): 구체적인 산출물
  - When (언제): 시간적 맥락
  - Where (어디서): 적용 환경
  - Why (왜): 목적과 배경
  - How (어떻게): 형식과 방법

### 시각 요소
- 5W1H를 원형 다이어그램으로 배치
- 각 요소에 아이콘 추가

### 발표자 노트

## 슬라이드 11: Markdown Before/After 비교
**타입**: 비교 슬라이드
**소요 시간**: 1분

### 콘텐츠
**제목**: "구조화의 힘: Before vs After"

**Before (좌측)** - 일반 텍스트 (배경 #FEE2E2):
```
논문 주제는 온라인 학습이고 연구 질문은
학습자들이 어떤 전략을 사용하는가와 효과적인
전략은 무엇인가입니다. 방법론은 설문조사
200명과 인터뷰 10명이고 기간은 3개월입니다.
```

**After (우측)** - Markdown (배경 #D1FAE5):
```markdown
# 연구 프로젝트

## 주제
온라인 학습 환경에서의 자기조절학습 전략

## 연구 질문
1. 학습자들이 사용하는 전략은?
2. 효과적인 전략의 특징은?

## 방법론
- **설문조사**: n=200
- **심층 인터뷰**: n=10
- **기간**: 3개월
```

### 시각 요소
- 좌우 비교 레이아웃
- Before는 빨간 배경, After는 녹색 배경
- 화살표로 개선 표시

### 발표자 노트

- "AI도 사람처럼 구조화된 정보를 더 잘 이해합니다"

## 슬라이드 14: 프롬프트 작성 원칙 3 - Few-shot Prompting
**타입**: 콘텐츠 슬라이드
**소요 시간**: 2분

### 콘텐츠
**제목**: "원칙 3: 예시 기반 학습 (Few-shot Prompting)"

**개념** (18pt):
AI에게 원하는 결과의 예시를 2-3개 제공하여
패턴을 학습하게 하는 기법

**언제 사용하나?** (16pt):
- 특정 형식이나 스타일을 유지하고 싶을 때
- 일관된 품질의 결과가 필요할 때
- 복잡한 작업을 설명하기 어려울 때

**효과** (16pt):
✓ 일관성 향상 (80% → 95%)
✓ 설명 시간 단축 (10분 → 2분)
✓ 오류 감소 (30% → 5%)

### 시각 요소
- 예시 → AI → 결과 플로우차트
- 통계는 강조색으로 표시

### 발표자 노트
- "사람도 예시를 보면 더 잘 이해하죠? AI도 마찬가지입니다"
- "특히 논문 초록, 이메일 등 형식이 중요한 작업에 효과적입니다"

## 슬라이드 15: Few-shot Prompting 예시
**타입**: 예시 슬라이드
**소요 시간**: 1분

### 콘텐츠
**제목**: "Few-shot Prompting 실전 예시"

**시나리오**: 논문 초록 작성

**프롬프트 구조** (14pt, 고정폭 폰트):
```markdown
# 과제
다음 논문의 초록을 작성해줘.

# 좋은 초록 예시 1
[예시 초록 1 - 150단어, 구조화됨]

# 좋은 초록 예시 2
[예시 초록 2 - 150단어, 구조화됨]

# 내 논문 정보
- 주제: [주제]
- 연구 방법: [방법]
```
