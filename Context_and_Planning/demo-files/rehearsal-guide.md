# 시연 리허설 가이드

이 문서는 효과적인 시연을 위한 리허설 가이드입니다.

## 🎯 리허설 목표

1. **시간 준수**: 각 시연을 정확히 5분 이내로
2. **자연스러움**: 대본을 읽는 것이 아닌 자연스러운 설명
3. **문제 대응**: 예상 문제에 대한 빠른 대응
4. **핵심 전달**: 각 시연의 핵심 메시지 명확히 전달

## 📅 리허설 일정 (강의 1주일 전)

### Day 1-2: 개별 시연 연습

**시연 1: VS Code (5회 반복)**
```markdown
# 연습 체크리스트

## 1회차
- [ ] 전체 흐름 파악
- [ ] 시간 측정: [X]분 [Y]초
- [ ] 막힌 부분 메모

## 2회차
- [ ] 설명 추가하며 진행
- [ ] 시간 측정: [X]분 [Y]초
- [ ] 말하는 속도 조절

## 3회차
- [ ] 청중에게 설명한다고 생각하고
- [ ] 시간 측정: [X]분 [Y]초
- [ ] 핵심 메시지 강조

## 4회차
- [ ] 문제 상황 시뮬레이션
- [ ] 백업으로 전환 연습
- [ ] 시간 측정: [X]분 [Y]초

## 5회차
- [ ] 최종 점검
- [ ] 자연스러운 흐름 확인
- [ ] 시간 측정: [X]분 [Y]초
- [ ] 목표: 4분 30초 - 5분
```

**다른 시연도 동일하게 5회씩 반복**

## 🎭 리허설 시 체크할 사항

### 말하기

**속도**:
- [ ] 너무 빠르지 않은가? (긴장하면 빨라짐)
- [ ] 너무 느리지 않은가? (지루함)
- [ ] 적절한 속도: 분당 120-150 단어

**명확성**:
- [ ] 발음이 명확한가?
- [ ] 전문 용어를 설명하는가?
- [ ] 청중이 이해할 수 있는 언어인가?

**에너지**:
- [ ] 목소리에 에너지가 있는가?
- [ ] 단조롭지 않은가?
- [ ] 중요한 부분에서 강조하는가?

### 동작

**마우스 사용**:
- [ ] 마우스 움직임이 천천히, 명확한가?
- [ ] 중요한 부분을 가리키는가?
- [ ] 불필요한 움직임은 없는가?

**화면 전환**:
- [ ] 전환이 부드러운가?
- [ ] 청중이 따라올 시간을 주는가?
- [ ] 갑작스러운 전환은 없는가?

### 시간 관리

**각 단계별**:
- [ ] 1단계: 목표 시간 내
- [ ] 2단계: 목표 시간 내
- [ ] 3단계: 목표 시간 내
- [ ] 4단계: 목표 시간 내
- [ ] 전체: 5분 ± 30초

**시간 조절**:
- 빠를 때: 설명 추가, 질문 받기
- 느릴 때: 마지막 단계 축약, 핵심만

- 설명 명확성: ⭐⭐⭐⭐⭐ (5점 만점)

## 💡 리허설 팁

### 효과적인 연습 방법

1. **녹화하기**
   - 자신의 시연을 녹화
   - 객관적으로 평가
   - 개선점 발견

2. **거울 보며 연습**
   - 표정 확인
   - 제스처 확인
   - 자신감 있는 모습

3. **타이머 사용**
   - 항상 시간 측정
   - 각 단계별 시간 파악
   - 조절 연습

4. **실제 환경 시뮬레이션**
   - 서서 연습
   - 큰 화면 사용
   - 청중이 있다고 상상

### 피해야 할 것

1. **과도한 연습**
   - 너무 많이 연습하면 기계적
   - 5회 정도가 적당
   - 자연스러움 유지

2. **완벽주의**
   - 작은 실수는 괜찮음
   - 회복하는 능력이 더 중요
   - 유머로 넘기기

3. **대본 암기**
   - 대본을 외우지 말 것
   - 핵심 포인트만 기억
   - 자연스럽게 설명

## 🎤 시연 중 자주 하는 실수와 해결

### 실수 1: 너무 빠른 진행

**증상**: 3분 만에 시연 끝남

**해결**:
- 각 단계마다 "보시다시피..." 설명 추가
- 청중 반응 확인 ("보이시나요?")
- 중요한 부분 반복 설명

### 실수 2: 너무 느린 진행

**증상**: 7분 이상 소요

**해결**:
- 마지막 단계 축약
- 핵심만 설명
- "자세한 내용은 자료 참고" 안내

### 실수 3: 기술적 용어 남발

**증상**: 청중이 이해 못 하는 표정

**해결**:
- 전문 용어 사용 시 즉시 설명
- 비유 사용 ("마치 ~처럼")
- 구체적 예시 제공

### 실수 4: 청중과 눈 맞춤 없음

**증상**: 화면만 보고 설명

**해결**:
- 주기적으로 청중 쳐다보기
- "이해되시나요?" 확인
- 청중 반응 살피기

## 📊 리허설 평가 기준

### 우수 (90점 이상)
- ✅ 시간: 4분 30초 - 5분 30초
- ✅ 모든 단계 완료
- ✅ 핵심 메시지 명확
- ✅ 설명 이해하기 쉬움
- ✅ 자연스러운 진행

### 양호 (80-89점)
- ✅ 시간: 4분 - 6분
- ✅ 주요 단계 완료
- ⚠️ 일부 설명 보완 필요
- ✅ 전반적으로 이해 가능

### 보통 (70-79점)
- ⚠️ 시간: 3분 - 7분
- ⚠️ 일부 단계 생략
- ⚠️ 설명 보완 필요
- ⚠️ 추가 연습 권장

### 미흡 (70점 미만)
- ❌ 시간: 3분 미만 또는 7분 이상
- ❌ 핵심 단계 누락
- ❌ 설명 불명확
- ❌ 재연습 필수

## 🌟 성공적인 시연을 위한 마인드셋

### 긍정적 자기 암시

**시연 전**:
- "나는 이 내용을 잘 알고 있다"
- "충분히 연습했다"
- "작은 실수는 괜찮다"
- "청중은 나를 응원한다"

**시연 중**:
- "천천히, 명확하게"
- "청중과 함께 한다"
- "즐기면서 한다"

**문제 발생 시**:
- "침착하게 대응한다"
- "백업이 준비되어 있다"
- "이것도 학습의 일부다"

### 실수 대응 마인드

**작은 실수** (말 더듬기, 클릭 실수):
- 무시하고 계속 진행
- 청중은 대부분 눈치채지 못함

**중간 실수** (단계 건너뛰기):
- "아, 중요한 부분을 빠뜨렸네요"
- 간단히 설명하고 계속

**큰 실수** (도구 작동 안 함):
- 침착하게 백업으로 전환
- 유머로 분위기 전환
- "이것이 백업의 중요성입니다"

## 🎓 리허설에서 배운 교훈 (일반적)

### 시간 관리
- 처음엔 항상 예상보다 김
- 3회차부터 안정화
- 5회차면 자연스러워짐

### 설명
- 대본 없이 설명하는 것이 더 자연스러움
- 핵심 포인트만 기억
- 청중 반응 보며 조절

### 기술
- 문제는 항상 발생함
- 백업이 생명
- 침착함이 가장 중요

### 자신감
- 연습할수록 자신감 증가
- 5회 연습이면 충분
- 과도한 연습은 오히려 역효과
