# Exercise 1 시연용 프롬프트 모음
## 문헌 조사 자동화

**목적**: Copilot 워크북 Exercise 1을 5분 내 시연  
**대상**: Part 2 Section 2.2 내용  
**파일**: literature-search-starter.md  

## Step 1: 컨텍스트 제공 (30초)

### 강사 액션
1. `literature-search-starter.md` 파일 열기
2. 주석이 이미 작성되어 있음을 확인
3. 주석 내용 설명

### 강사 멘트
"먼저 주석으로 연구 컨텍스트를 제공합니다.
연구 주제, 목적, 방법, 기간, 전공 정보를 포함했습니다.
Copilot은 이 정보를 바탕으로 적절한 제안을 합니다."

### 예상 화면
- 주석 블록이 보임
- 커서가 주석 아래 "# 문헌 조사" 위치

## Step 2: 검색 키워드 생성 (2분)

### 강사 액션
1. "# 문헌 조사" 입력 후 Enter
2. Tab 키 → Copilot이 "## 검색 키워드" 제안 예상
3. 수락 후 다음 줄에 "### 한글 키워드" 입력
4. "- " 입력 후 Tab → Copilot이 키워드 제안
5. 5-7개 키워드 수락 (Tab 반복)
6. "### English Keywords" 입력 후 동일 과정

### 강사 멘트
"보시다시피 Copilot이 연구 주제와 관련된 키워드를 제안합니다.
Tab으로 수락, Esc로 거부할 수 있습니다.
마음에 들지 않으면 직접 작성하면 됩니다.
한글과 영문 키워드를 모두 생성하여 데이터베이스 검색에 활용합니다."

### 예상 Copilot 제안
**한글 키워드**:
- 온라인 학습
- 학습자 참여도
- 원격 교육
- 이러닝
- 학습 동기
- 자기조절학습
- online learning
- learner engagement
- student participation
- e-learning
- learning motivation
- self-regulated learning

### 트러블슈팅
- **제안 안 나옴** → Tab 키 여러 번 눌러보기
- **엉뚱한 제안** → Esc로 거부하고 직접 입력
- **시간 부족** → 3-4개만 시연하고 "이런 식으로 계속" 설명

## Step 3: 논문 요약 템플릿 생성 (1.5분)

### 강사 액션
1. "## 논문 요약 템플릿" 입력
2. "### 논문 1" 입력 후 Enter
3. Tab → Copilot이 "**기본 정보**" 제안 예상
4. 수락하고 계속 진행
5. 주요 섹션 (기본 정보, 연구 목적, 연구 방법, 주요 결과, 관련성) 생성

### 강사 멘트
"이제 논문 요약 템플릿을 만듭니다.
Copilot이 학술 논문 요약의 표준 구조를 제안합니다.
이 템플릿을 사용하면 모든 논문을 일관된 형식으로 정리할 수 있습니다."

### 예상 Copilot 제안
- **기본 정보** (제목, 저자, 연도, 저널)
- **연구 목적**
- **연구 방법** (설계, 대상, 도구, 분석)
- **주요 결과**
- **본 연구와의 관련성**

## Step 4: 마무리 및 다음 단계 (30초)

### 강사 멘트
"이렇게 Copilot으로 문헌 조사의 기본 구조를 빠르게 만들 수 있습니다.
Exercise 2-4는 자료로 제공되니 직접 실습해보세요.
- Exercise 2: 연구 계획서 작성
- Exercise 3: 데이터 분석 스크립트
- Exercise 4: 논문 작성 지원

자세한 내용은 Part 2 자료를 참고하세요."

### 다음 시연 연결
"이제 MCP 설치 방법을 보여드리겠습니다."

## ⏱️ 타이밍 breakdown

- **Step 2**: 2분 (키워드 생성)
- **Step 3**: 1.5분 (템플릿 생성)
- **Step 4**: 30초 (마무리)
- **Buffer**: 30초
- **Total**: 5분

## 🚨 긴급 상황 대응

### Copilot 제안이 전혀 안 나올 때
1. "이 강의실에서 Copilot이 잘 작동하지 않네요"
2. 완성된 `literature-search-completed.md` 파일 열기
3. "이것이 일반적인 Copilot 제안 결과입니다" 설명

### 제안이 부적절할 때
1. "이 제안은 우리 연구와는 맞지 않네요"
2. Esc 키로 거부하는 동작 보이기
3. "이럴 때는 직접 입력을 하거나 다른 제안을 기다리면 됩니다"

### 시간 초과 시
1. Step 2에서 키워드 3-4개만 시연
2. "이런 식으로 계속 진행할 수 있습니다" 설명
3. 완성된 파일로 템플릿 구조 설명

## 🎯 핵심 확인 사항

- **컨텍스트 제공**: 주석으로 연구 정보 명확히
- **제안 수락**: Tab 키 사용법 명확히
- **제안 거부**: Esc 키 사용법 설명
- **일관성**: 동일한 형식 유지
- **대안 도구**: Copilot Chat, 직접 작성 언급
