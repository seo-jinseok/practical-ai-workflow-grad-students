# MCP 설치 확인 테스트 프롬프트

**목적**: MCP 서버가 정상 작동하는지 확인  
**사용 시점**: 설치 완료 후, 시연 Step 3-4  

## Test 1: 서버 인식 확인

### 프롬프트
```
task-master-mcp를 사용할 수 있나요?
```

### 예상 응답 (성공)
```
네, task-master-mcp를 사용할 수 있습니다.
이 도구는 프로젝트 관리와 작업 추적을 도와드립니다.
어떤 프로젝트를 시작하시겠습니까?
```

### 예상 응답 (실패)
```
죄송합니다. task-master-mcp를 찾을 수 없습니다.
```
→ 설정 파일 재확인 필요

## 🔧 빠른 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| "찾을 수 없습니다" | MCP 미연결 | 설정 파일 확인, Claude 재시작 |
| 응답 없음 | 네트워크 지연 | 30초 대기, 재시도 |
| 오류 메시지 | JSON 문법 오류 | VS Code에서 빨간 밑줄 확인 |
| 일부만 작동 | 서버별 설정 필요 | 각 서버 문서 참조 |

## 🎯 시연 목표

### 학생들이 이해해야 할 것
1. **MCP 설치 개념**: 설정 파일 편집으로 도구 연결
2. **확인 방법**: 간단한 테스트 프롬프트로 작동 확인
3. **대안 도구**: 설치가 어려우면 다른 옵션 사용 가능
4. **실용성**: 도구보다 접근 방식이 더 중요
