**전환 멘트** (슬라이드 → 시연):

### Demo 2: GitHub Copilot (70-75분, 5분)

**전환 멘트**:

**시연 진행**:

### Demo 5: Copilot 워크북 (75-80분, 5분)

**전환 멘트** (Demo 2 → Demo 5):
"방금 본 Copilot을 연구에 어떻게 적용하는지 보여드리겠습니다.
Part 2 Copilot 워크북 Exercise 1입니다."

**시연 진행**:
[Demo 5 스크립트 참조: `05-copilot-workbook/exercise1-prompts.md`]

**Step 1**: 주석으로 컨텍스트 제공 (30초)  
**Step 2**: 키워드 생성 (2분)  
**Step 3**: 템플릿 생성 (1.5분)  
**Step 4**: 마무리 (30초)

**시연 → 슬라이드 전환**:
"이렇게 Copilot으로 문헌 조사를 체계화할 수 있습니다.
Exercise 2-4는 자료로 제공되니 직접 실습해보세요.

이제 MCP와 SpecKit을 소개하겠습니다."

### Demo 8: 통합 워크플로우 (100-105분, 5분)

**전환 멘트** (슬라이드 → 시연):
"지금까지 배운 모든 것을 통합하여 실제 프로젝트에 적용하는 방법을 보여드립니다.
6개월 석사논문 프로젝트를 5분으로 압축했습니다."

**시연 진행**:

**시연 → 슬라이드 전환**:
"이것이 Part 1-2-3를 모두 통합한 실제 워크플로우입니다.

1. Context Engineering (Part 1)

이제 마무리하겠습니다."

## v13.0 새로운 시연의 특징

### Demo 5: Copilot 워크북
- **참가자 대상**: 연구 경험이 있는 학생
- **시연 도구**: VS Code + GitHub Copilot
- **핵심 학습**: AI와 협업하는 방법
- **시간 배분**: 실습 위주 (2-3분 실습 + 1-2분 설명)

### Demo 6: MCP 설치 (선택적)
- **참가자 대상**: 기술에 익숙한 학생
- **시연 도구**: Claude Desktop + 설정 파일
- **핵심 학습**: 도구 설정의 원리
- **주의사항**: 대안 도구 반드시 언급

### Demo 7: SpecKit (선택적)
- **참가자 대상**: 프로젝트 관리 경험 있는 학생
- **시연 도구**: Terminal + CLI
- **핵심 학습**: 체계적 프로젝트 관리
- **주의사항**: "선택적 도구" 명시적 강조

### Demo 8: 통합 워크플로우
- **참가자 대상**: 전체 과정을 이해하고 싶은 학생
- **시연 도구**: 모든 도구 통합
- **핵심 학습**: 전체 프로세스 이해
- **중요성**: 가장 중요한 시연 (압축 수업)
