# 원칙 1: 명확성과 구체성

## 5W1H 원칙
- **Who**: 대상 독자/사용자
- **What**: 구체적인 산출물  
- **When**: 시간적 맥락
- **Where**: 적용 환경
- **Why**: 목적과 배경
- **How**: 형식과 방법

## ❌ 나쁜 예시
```
논문 써줘
```

## ✅ 좋은 예시
```
교육학 분야 질적 연구 방법론에 관한
석사 논문의 서론 초안을 작성해줘.

연구 주제: '온라인 학습 환경에서 학습자의 자기조절학습 전략'
분량: 3-4페이지, APA 7판 스타일
```

# 원칙 2: Markdown 활용

## 왜 Markdown인가?
- ✓ AI가 이해하기 쉬운 명확한 계층 구조
- ✓ 재사용 가능 (파일로 저장)
- ✓ 버전 관리 용이 (Git 호환)
- ✓ 플랫폼 독립적
- ✓ 5분이면 기본 습득

## 기본 문법
```markdown
# 제목 1
## 제목 2
**굵은 글씨**
- 목록 항목
1. 번호 목록
> 인용문
```

# 구조화의 힘: Before vs After

## Before - 일반 텍스트
```
논문 주제는 온라인 학습이고 연구 질문은 학습자들이 어떤 전략을 사용하는가와 
효과적인 전략은 무엇인가입니다. 방법론은 설문조사 200명과 인터뷰 10명이고 
기간은 3개월입니다.
```

## After - Markdown
```markdown
# 연구 프로젝트

## 주제
온라인 학습 환경에서의 자기조절학습 전략

## 연구 질문
1. 학습자들이 사용하는 전략은?
2. 효과적인 전략의 특징은?

## 방법론
- **설문조사**: n=200
- **심층 인터뷰**: n=10
- **기간**: 3개월
```

# MCP (Model Context Protocol)

## 정의
AI 모델에 외부 컨텍스트(데이터, 도구, 지식)를 표준화된 방식으로 제공하는 오픈 프로토콜

## 2025년 10월 현재 상황
- Anthropic이 주도하는 오픈 표준
- GitHub에서 MCP 서버로 전환 중 (2025년 11월 완전 전환)
- 다양한 AI 플랫폼에서 표준으로 채택

## 핵심 개념
- **확장성**: AI의 능력을 무한히 확장
- **표준화**: 일관된 방식으로 통합  
- **모듈화**: 필요한 도구만 선택
- **호환성**: 한 번 구축하면 여러 AI에서 사용

# Spec-driven Planning이란?

## 정의
프로젝트의 **목표, 범위, 요구사항**을 명확히 정의한 문서(Spec)를 기반으로 계획을 수립하고 실행하는 방법론

## 핵심 요소
- **Spec (명세서)**: 프로젝트의 청사진
- **문서화**: 모든 것을 기록
- **추적 가능성**: 변경 이력 관리
- **AI 친화적**: 명확한 입력 → 명확한 출력

## 전통적 방식 vs Spec-driven
- 전통적: 머릿속 계획 → 방향 상실 → 재작업
- Spec-driven: 문서화된 계획 → 명확한 방향 → 체계적 완료

# 작업 분해: Work Breakdown Structure

"논문 쓰기"는 너무 크고 막연한 작업

- 명확하고 구체적으로 (5W1H)
- Markdown으로 구조화
- MCP 도구 활용

**"좋은 질문이 좋은 답을 만든다"**

## 2. Planning & Task Management 📋
- Spec 문서로 명확한 계획
- 관리 가능한 단위로 분해
- 3단계 검증 (AI + 자가 + 전문가)

**"명확한 계획이 성공적인 실행을 만든다"**

## 3. Integrated Workflow 🔄
- Context 설정 → Planning → 실행 → 검증
- 각 단계에서 AI와 협업
- 지속적인 대화와 개선

**"AI는 도구가 아니라 협업 파트너"**

# 오늘부터 시작하는 실천 가이드

## Level 1: 오늘 저녁 (30분)
1. 현재 프로젝트를 Markdown으로 정리
2. AI에게 첫 질문: "이 연구 프로젝트를 검토하고 개선점을 제안해줘"
3. 배운 내용 복습

## Level 2: 이번 주 (2시간)
1. Spec 문서 작성 (목표, 범위, 일정, 산출물)
2. 작업 분해 요청: "이 프로젝트를 완료 가능한 작업들로 분해해줘"
3. 첫 작업 완료 (가장 작은 작업 하나를 AI와 함께)

## Level 3: 이번 달 (지속적)
1. 주간 루틴 확립 (매주 금요일 진행 상황 점검)
2. 검증 습관화 (모든 AI 출력물 검토)
3. 워크플로우 개선 (자신에게 맞게 조정)

**1개월 후, 여러분의 연구 방식이 완전히 달라져 있을 것입니다!**
