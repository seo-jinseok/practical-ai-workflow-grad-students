# 도구 시연 계획 및 통합 가이드

## 시연 도구 개요

### 선정된 도구
1. **VS Code**: 텍스트 편집기 및 개발 환경 (무료)
2. **GitHub Copilot**: AI 기반 코드/문서 자동 완성 (학생 무료)
3. **Claude.ai / ChatGPT**: AI 기반 문서/코드 생성 (무료)
4. **GitHub Projects**: 프로젝트 관리 및 작업 추적 (무료)
5. **task-master-mcp**: AI 기반 작업 관리 시스템 (오픈소스)

### 시연 원칙
- **도구 중립성 유지**: 특정 도구에 종속되지 않는 개념 강조
- **실용성 중심**: 대학원생이 즉시 활용 가능한 기능
- **짧고 명확하게**: 각 시연 5분 이내
- **백업 준비**: 라이브 시연 실패 시 녹화 영상 준비

## 시연 1: VS Code + Markdown (Part 1 중반, 5분)

### 시연 위치
- **섹션**: Part 1 - Context Engineering
- **타이밍**: 프롬프트 작성 원칙 설명 후 (강의 시작 후 약 25분)
- **전환 멘트**: "이제 실제로 어떻게 구조화된 프롬프트를 작성하는지 보여드리겠습니다."

### 시연 시나리오

#### 준비 사항
- VS Code 실행 (깨끗한 워크스페이스)
- Markdown Preview Enhanced 확장 설치
- 예제 파일: `research-prompt.md` (빈 파일)
- 화면 공유 설정 (글씨 크기 18pt 이상)

#### 시연 단계 (5분)

**1단계: 기본 Markdown 작성 (2분)**
```markdown
# 연구 프로젝트: 온라인 학습 환경 분석

## 연구 배경
- 코로나19 이후 온라인 학습 급증
- 학습 효과성에 대한 우려

## 연구 질문
1. 온라인 학습의 효과적인 요소는 무엇인가?
2. 학습자 만족도에 영향을 미치는 요인은?

## 요청사항
위 연구 주제에 대한 문헌 고찰 개요를 작성해줘.
다음 구조로 작성:
- 주요 이론적 배경 (3-4개)
- 선행 연구 동향 (최근 5년)
- 연구 갭 (gap) 분석
```

**설명 포인트**:
- 제목(#), 부제목(##)으로 계층 구조
- 목록(-, 1.)으로 항목 정리
- 명확한 요청사항 명시

**2단계: Preview 기능 활용 (1분)**
- `Cmd/Ctrl + K V`로 미리보기 열기
- 구조화된 문서의 가독성 강조
- "AI가 이런 구조를 더 잘 이해합니다"

**3단계: AI에 입력 (1.5분)**
- 작성한 Markdown을 복사
- ChatGPT 또는 Claude 웹 인터페이스에 붙여넣기
- AI 응답 확인 (미리 준비된 응답 사용 가능)
- "구조화된 입력이 구조화된 출력을 만듭니다"

**4단계: 결과 저장 및 버전 관리 (0.5분)**
- AI 응답을 같은 파일에 추가
- 파일 저장
- "이렇게 모든 프롬프트와 응답을 문서로 관리할 수 있습니다"

### 강조 메시지
- Markdown은 배우기 쉽고 강력한 도구
- 구조화가 AI 소통의 핵심
- VS Code는 무료이며 다양한 확장 기능 제공

### 백업 계획
- 미리 녹화한 시연 영상 (3분 버전)
- 스크린샷 슬라이드 (단계별 캡처)

### 시연 위치

### 시연 시나리오

#### 준비 사항
- VS Code + GitHub Copilot 확장 설치 및 로그인
- 예제 파일: `research-plan.md` (빈 파일)
- Copilot 활성화 확인
- **학생 무료 안내**: GitHub Student Developer Pack으로 무료 사용 가능

#### 시연 단계 (5분)

**1단계: 주석으로 요구사항 작성 (1.5분)**
```markdown
<!--
석사 논문 연구 계획서
주제: 온라인 학습 환경에서의 학습자 참여도 분석
기간: 6개월
방법론: 혼합 연구 (설문 + 인터뷰)
-->

# 연구 계획서

## 1. 연구 제목
```

**설명 포인트**:
- 주석으로 컨텍스트 제공
- Copilot이 이를 바탕으로 제안 생성

**2단계: Copilot 제안 수락 (2분)**
- 제목 입력 시작하면 Copilot이 자동 완성 제안
- Tab 키로 수락
- 다음 섹션(## 2. 연구 배경) 입력 시작
- Copilot이 내용 제안
- 계속 Tab으로 수락하며 빠르게 문서 완성

**실시간 설명**:
- "보시다시피 Copilot이 문맥을 이해하고 적절한 내용을 제안합니다"
- "마음에 들지 않으면 무시하고 직접 작성하면 됩니다"
- "이렇게 초안을 빠르게 작성하고 나중에 수정할 수 있습니다"

**3단계: 코드 예시 (선택적, 1분)**
- Python 데이터 분석 코드 작성 시연 (이공계 학생 대상)
```python
# 설문 데이터 분석: 학습자 참여도와 만족도 상관관계
import pandas as pd
import matplotlib.pyplot as plt

# 데이터 로드
```
- Copilot이 나머지 코드 제안
- "연구 방법론 구현도 AI가 도와줍니다"

**4단계: 마무리 (0.5분)**
- "Copilot은 문서 작성뿐 아니라 코드, 이메일, 보고서 등 모든 텍스트 작업에 활용 가능"
- "학생은 GitHub Student Developer Pack으로 무료 사용 가능 (2025년 10월 기준)"
- "신청 방법: education.github.com에서 학생 인증"

### 강조 메시지
- Spec-driven 접근: 명확한 요구사항이 좋은 결과를 만듦
- 생산성 향상: 초안 작성 시간 50% 이상 단축
- 무료 접근성: 학생 신분으로 무료 사용
- 학습 도구: AI 제안을 보며 좋은 문서 구조 학습

### 백업 계획
- 미리 작성된 예제 파일 (Copilot 제안 결과)
- 녹화 영상 (2분 버전)
- Claude.ai 또는 ChatGPT로 대체 시연 가능

## 시연 3: GitHub Projects + SPEC 문서 (Part 2 후반, 5분)

### 시연 위치
- **섹션**: Part 2 - Planning & Task Management
- **타이밍**: 작업 분해 방법론 설명 후 (강의 시작 후 약 80분)
- **전환 멘트**: "이제 작성한 Spec을 실제 프로젝트 관리로 연결해봅시다."

### 시연 시나리오

#### 준비 사항
- GitHub 계정 및 테스트 저장소
- GitHub Projects 활성화 및 설정
- 예제 프로젝트: "석사 논문 작성 프로젝트"

#### 시연 단계 (5분)

**1단계: Spec 문서 작성 (1.5분)**
- GitHub 저장소에 `SPEC.md` 파일 생성
```markdown
# 석사 논문 프로젝트 명세

## 목표
온라인 학습 환경에서의 학습자 참여도 분석 논문 완성

## 범위
- 문헌 고찰 (50편 이상)
- 설문 조사 (n=200)
- 심층 인터뷰 (n=10)
- 논문 작성 (80-100페이지)

## 일정
- 시작: 2025년 3월
- 완료: 2025년 8월
- 중간 발표: 2025년 6월

## 산출물
- [ ] 연구 계획서
- [ ] IRB 승인
- [ ] 문헌 고찰 초안
- [ ] 설문 도구 개발
- [ ] 데이터 수집
- [ ] 데이터 분석
- [ ] 논문 초안
- [ ] 최종 논문
```

**2단계: GitHub Projects로 작업 분해 (2분)**
- GitHub Projects에서 작업 분해:
- GitHub Issues와 연결하여 작업 생성
- 각 Issue에 라벨, 마일스톤, 예상 시간 자동 할당
- 의존성 관계 시각화

**화면 설명**:
- "Spec 문서 하나로 전체 프로젝트가 관리 가능한 작업들로 분해되었습니다"
- "각 작업의 우선순위와 의존성이 자동으로 설정되었습니다"

**3단계: 프로젝트 보드 확인 (1분)**
- GitHub Projects 보드 열기
- Kanban 보드에 작업들이 자동 배치
- To Do, In Progress, Done 컬럼
- "시각적으로 진행 상황을 한눈에 파악할 수 있습니다"

**4단계: 작업 추적 (0.5분)**
- 하나의 Issue를 "In Progress"로 이동
- 체크리스트 업데이트
- "실제 작업하면서 진행 상황을 업데이트합니다"

### 강조 메시지
- Spec 문서가 프로젝트 관리의 중심
- GitHub의 기본 기능으로 별도 설치 불필요
- 협업 시 특히 유용 (지도교수, 동료와 공유)

### 설치/설정 안내
GitHub Projects는 GitHub의 기본 기능으로 별도 설치가 필요하지 않습니다:
1. 저장소 페이지에서 Projects 탭 클릭
2. "New project" 버튼으로 새 프로젝트 생성
3. Template 선택 (Board, Table, Roadmap 중 선택)
4. 프로젝트에 Issues 연결하여 작업 관리

### 대안 도구 언급
- "GitHub 외에도 Notion, Trello, Asana 등 다양한 도구 활용 가능"
- "중요한 것은 도구가 아니라 Spec-driven 접근 방식"

### 백업 계획
- 미리 설정된 프로젝트 보드 스크린샷
- 녹화 영상 (3분 버전)

## 시연 4: task-master-mcp (Part 3 중반, 5분)

### 시연 위치
- **섹션**: Part 3 - 실전 워크플로우 통합
- **타이밍**: End-to-End 워크플로우 시연 중 (강의 시작 후 약 100분)
- **전환 멘트**: "이제 AI가 직접 프로젝트 매니저 역할을 하는 것을 보여드리겠습니다."

### 시연 시나리오

#### 준비 사항
- task-master-mcp 설치 및 설정
- Claude Desktop 또는 호환 AI 클라이언트
- 예제 프로젝트: 진행 중인 논문 작성 프로젝트

#### 시연 단계 (5분)

**1단계: 프로젝트 초기화 (1분)**
- AI 클라이언트에서 `task-master-mcp` 활성화
- 프롬프트 입력:
```
새 프로젝트를 시작합니다.
프로젝트명: 석사 논문 - 온라인 학습 참여도 분석
기간: 6개월
현재 상태: 문헌 고찰 진행 중 (30% 완료)

다음 작업을 제안해주세요.
```

**AI 응답 (미리 준비)**:
- 프로젝트 구조 생성
- 현재 진행 상황 분석
- 다음 우선순위 작업 3개 제안
- 예상 소요 시간 및 마감일

**2단계: 작업 추가 및 관리 (2분)**
- 프롬프트: "문헌 고찰을 완료하기 위한 세부 작업을 추가해줘"
- AI가 자동으로 작업 분해:
  - 추가 논문 10편 검토
  - 주제별 분류 및 요약
  - 연구 갭 분석
  - 문헌 고찰 초안 작성
- 각 작업에 체크박스 및 예상 시간 할당

**화면 설명**:
- "AI가 프로젝트 컨텍스트를 이해하고 적절한 작업을 제안합니다"
- "작업 간 의존성도 자동으로 파악합니다"

**3단계: 진행 상황 업데이트 (1.5분)**
- 프롬프트: "논문 5편 검토를 완료했어. 진행 상황을 업데이트하고 다음 할 일을 알려줘."
- AI 응답:
  - 작업 완료 체크
  - 전체 진행률 업데이트 (30% → 35%)
  - 다음 우선순위 작업 제안
  - 일정 지연 여부 확인
  - 필요시 일정 조정 제안

**실시간 설명**:
- "AI가 프로젝트 매니저처럼 진행 상황을 추적합니다"
- "막힌 부분이 있으면 해결 방법도 제안합니다"

**4단계: 통합 워크플로우 시연 (0.5분)**
- 프롬프트: "다음 논문 검토를 위한 구조화된 프롬프트를 작성해줘"
- AI가 Part 1에서 배운 Context Engineering 기법 적용
- "이렇게 모든 것이 하나의 워크플로우로 통합됩니다"

### 강조 메시지
- AI를 단순 도구가 아닌 협업 파트너로 활용
- Context Engineering + Planning의 실제 통합
- 지속적인 대화를 통한 프로젝트 관리

### MCP 개념 재강조
- "task-master-mcp는 MCP를 활용한 예시입니다"
- "다양한 MCP 도구들을 조합하여 자신만의 워크플로우 구축 가능"
- "중요한 것은 도구가 아니라 체계적 접근 방식"

### 백업 계획
- 미리 준비된 대화 스크립트
- 스크린샷 시퀀스
- 녹화 영상 (4분 버전)

## 시연 실패 대응

### 일반적 문제 및 해결

**인터넷 연결 끊김**
- 즉시 백업 영상으로 전환
- "미리 준비한 영상으로 보여드리겠습니다"

**도구 오작동**
- 침착하게 재시도 (1회)
- 실패 시 스크린샷으로 설명
- "라이브 시연의 위험이죠. 실제로는 이렇게 작동합니다"

**AI 응답 지연**
- "AI가 생각하는 동안 질문 받겠습니다"
- 30초 이상 지연 시 미리 준비한 응답 사용

### 유머로 전환
- "AI도 가끔 쉬고 싶나 봅니다"
- "이것이 바로 백업 계획의 중요성입니다"
- "실전에서는 이런 일이 생기죠. 당황하지 마세요"

## 시연 준비 체크리스트

### 1주일 전
- [ ] 모든 도구 설치 및 테스트
- [ ] 예제 파일 및 프로젝트 준비
- [ ] 백업 영상 녹화 (각 시연별)
- [ ] 스크린샷 캡처 및 슬라이드 삽입

### 1일 전
- [ ] 강의실 인터넷 연결 확인
- [ ] 프로젝터 해상도 테스트
- [ ] 모든 도구 최신 버전 업데이트
- [ ] 예제 파일 초기화 (깨끗한 상태)

### 강의 당일 (1시간 전)
- [ ] 노트북 완충
- [ ] 모든 도구 실행 테스트
- [ ] 백업 USB 준비
- [ ] 화면 공유 설정 확인
- [ ] 글씨 크기 확대 (18pt 이상)

### 각 시연 직전
- [ ] 해당 도구 실행 확인
- [ ] 예제 파일 준비 상태 확인
- [ ] 백업 영상 준비 (클릭 한 번에 재생 가능)
- [ ] 타이머 설정 (5분)

## 시연 후 피드백 수집

### 즉각적 피드백
- "이 도구를 사용해보고 싶으신 분?" (손들기)
- "어떤 부분이 가장 유용해 보이나요?" (2-3명 발표)

### 강의 후 설문
- 각 도구의 유용성 평가 (1-5점)
- 실제 사용 의향
- 추가로 보고 싶은 도구

### 후속 지원
- 도구 설치 가이드 문서 제공
- 질문 받을 이메일/채널 안내
- 추가 튜토리얼 링크 공유
