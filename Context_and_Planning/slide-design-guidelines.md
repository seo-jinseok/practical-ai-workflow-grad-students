## 📖 템플릿 사용법

### 1. 템플릿 적용하기

#### PowerPoint 사용법
1. `lecture-template.pptx` 파일을 다운로드하여 엽니다
2. "디자인" 탭에서 "테마"를 확인합니다
3. 기존 슬라이드에 적용하려면: "디자인" → "테마" → "찾아보기"에서 템플릿 선택
4. 새 슬라이드로 만들려면: "홈" → "새 슬라이드" → 마스터 레이아웃 선택

#### Keynote 사용법
1. `lecture-template.key` 파일을 다운로드하여 엽니다
2. "보기" → "편집 마스터 슬라이드"에서 마스터 확인
3. 새 슬라이드로 만들려면: 도구 모음에서 마스터 레이아웃 선택

#### Google Slides 사용법
1. 제공된 링크로 템플릿을 엽니다
2. "파일" → "사본 만들기"로 개인 사본 생성
3. "슬라이드" → "마스터 편집"에서 마스터 확인
4. "새 슬라이드"에서 레이아웃 선택

### 2. 마스터 레이아웃 활용하기

#### 타이틀 레이아웃
- 강의 제목 (32-40pt, Bold)
- 부제목 (20-24pt, Medium)
- 강사 정보 및 날짜 (14-16pt, 하단 우측)

#### 콘텐츠 레이아웃
- 섹션 제목 (24-28pt, 좌측 상단)
- 주요 포인트 (18-20pt, bullet points)
- 상세 설명 (16-18pt)

#### 비교 레이아웃
- 좌우 분할 구조 활용
- 색상으로 구분 (좌측: 밝은 배경, 우측: 흰색 배경)
- 항목별 비교 시 색상 코드 사용

#### 데모 레이아웃
- 스크린샷 배치 영역 활용
- 단계 번호 명확히 표시
- 중요한 부분에 색상 박스 적용

### 3. 색상 및 폰트 커스터마이징

#### 색상 변경하기
- 마스터 슬라이드에서 색상 팔레트 수정
- 대학/기관 CI 색상으로 변경 가능
- 주요 색상만 변경하고 보조 색상은 유지 추천

#### 폰트 변경하기
- 시스템에 폰트가 설치되어 있어야 함
- 'Inter'와 'Noto Sans KR'이 설치되지 않은 경우 대체 폰트 사용
- 코드 예시에는 반드시 고정폭 폰트 사용

### 4. 고급 활용 팁

#### 애니메이션 추가
- 각 요소별로 자연스러운 페이드인 효과
- 너무 복잡한 애니메이션은 피함
- 발표 시간 고려하여 타이밍 설정

#### 이미지 및 도표 활용
- 해상도 1920x1080 이상 권장
- PNG 형식으로 투명 배경 활용
- 파일 크기 50MB 이하 유지

#### 발표자 노트 작성
- 각 슬라이드마다 상세 설명 작성
- 타이밍 표시 (예: "이 슬라이드 2분")
- 주요 포인트 키워드 정리

## 7. 슬라이드 타입별 상세 가이드라인

### 1. 타이틀 슬라이드
**사용법**: 각 섹션/Part 시작, 강의 전체 타이틀
- **제목**: 32-40pt, Bold, 중앙 정렬
- **부제목**: 20-24pt, Medium, 중앙 정렬
- **메타 정보**: 14-16pt, Regular, 하단 우측 정렬
- **배경**: 그라데이션 또는 솔리드 컬러 사용 가능
- **주의사항**: 너무 많은 텍스트 피함, 시각적 여백 40% 이상

### 2. 콘텐츠 슬라이드
**사용법**: 개념 설명, 원칙 소개, 일반적인 정보 전달
- **제목**: 24-28pt, 좌측 상단 정렬
- **주요 포인트**: 18-20pt, bullet points 또는 번호 목록
- **설명 텍스트**: 16-18pt, 충분한 행간 확보
- **시각 요소**: 아이콘, 작은 다이어그램 활용
- **레이아웃**: 좌측 제목 + 우측 콘텐츠, 또는 상하 구분

### 3. 비교 슬라이드
**사용법**: Before/After, 장점/단점, 전통적 방식 vs AI 방식 비교
- **레이아웃**: 좌우 분할 또는 상하 분할
- **색상 구분**: 좌측 #F9FAFB (밝은 회색), 우측 #FFFFFF (흰색)
- **제목**: 각각 "기존 방식" vs "제안 방식"
- **항목별 비교**: 색상으로 구분 (빨강=단점, 녹색=장점)
- **시각 요소**: 화살표, 체크 표시, 엑스 표시 활용

### 4. 데모/시연 슬라이드
**사용법**: 도구 사용법 설명, 라이브 시연 안내
- **스크린샷**: 실제 도구 화면 캡처 (해상도 1920x1080)
- **단계 번호**: 1→2→3→4 명확한 순서 표시
- **콜아웃**: 중요한 부분에 색상 박스나 화살표
- **코드 예시**: 구문 하이라이트, 복사하기 쉽게
- **백업 계획**: "실시간 데모 실패 시 녹화 영상 재생" 명시

### 5. 요약 슬라이드
**사용법**: 섹션 마무리, 핵심 내용 정리
- **핵심 포인트**: 3-5개 주요 takeaway
- **시각적 요소**: 번호 매기기, 체크 표시
- **다음 섹션 예고**: "다음으로는... 살펴보겠습니다"
- **색상**: 주요 포인트는 강조색상 사용

# 슬라이드 템플릿 및 디자인 가이드라인

## 1. 전체 슬라이드 구조

### 예상 슬라이드 수: 약 60-80장
- 도입부: 8-10장
- Part 1 (Context Engineering): 20-25장
- Part 2 (Planning & Task Management): 20-25장
- Part 3 (실전 워크플로우): 10-15장
- 마무리: 3-5장

### 슬라이드 타입 분류
1. **타이틀 슬라이드**: 각 섹션 시작
2. **개념 설명 슬라이드**: 이론 및 원칙 소개
3. **예시 슬라이드**: 실제 사례 및 코드/텍스트 예시
4. **비교 슬라이드**: Before/After, Good/Bad 비교
5. **프로세스 슬라이드**: 단계별 워크플로우
6. **시연 슬라이드**: 도구 사용 화면 캡처 또는 라이브 시연 안내
7. **요약 슬라이드**: 섹션별 핵심 정리
8. **인터랙션 슬라이드**: 질문, 토론, 실습 안내

## 2. 디자인 원칙

### 시각적 일관성
- **색상 팔레트**: 3-4가지 주요 색상으로 제한
  - 주색상: 신뢰감을 주는 블루 계열 또는 대학 CI 색상
  - 강조색: 주황/빨강 계열 (중요 포인트)
  - 보조색: 회색 계열 (텍스트, 배경)
  - 성공/긍정: 녹색 계열
- **폰트**: 2가지 폰트 패밀리
  - 제목: 고딕체 (굵고 명확한 폰트)
  - 본문: 명조체 또는 산세리프 (가독성 높은 폰트)
  - 코드/예시: 고정폭 폰트 (Consolas, Monaco 등)

### 레이아웃 원칙
- **여백 활용**: 슬라이드의 최소 20-30%는 여백으로 유지
- **시각적 계층**: 제목 > 부제목 > 본문 > 캡션 순으로 크기 차별화
- **정렬**: 좌측 정렬 기본, 중앙 정렬은 타이틀 슬라이드에만 사용
- **그리드 시스템**: 2열 또는 3열 레이아웃 활용

### 가독성 최적화
- **텍스트 양**: 슬라이드당 최대 6-7줄, 각 줄당 최대 10-12단어
- **글자 크기**: 제목 32-44pt, 본문 18-24pt, 캡션 14-16pt
- **대비**: 배경과 텍스트 간 충분한 명도 대비 확보
- **애니메이션**: 최소화, 필요시 페이드인 정도만 사용

## 3. 섹션별 슬라이드 템플릿

### 도입부 슬라이드 구성
1. **오프닝 슬라이드**: 강의 제목, 강사 정보, 날짜
2. **아이스브레이커**: 흥미로운 통계나 질문
3. **강의 목표**: 3-4개 핵심 학습 목표
4. **강의 로드맵**: 시간대별 주제 개요
5. **청중 배경 확인**: 간단한 설문 또는 손들기
6. **생성형 AI 현황**: 최신 트렌드 및 통계
7. **대학원 연구 활용 사례**: 실제 성공 사례 2-3개
8. **전환 슬라이드**: Part 1으로 이동

### Part 1 슬라이드 구성 (Context Engineering)
1. **섹션 타이틀**: "Part 1: Context Engineering"
2. **학습 목표**: 이 섹션에서 배울 4가지
3. **도입 질문**: "좋은 프롬프트란 무엇인가?"
4. **프롬프트 작성 원칙 1**: 명확성과 구체성 (개념 + 예시)
5. **프롬프트 작성 원칙 2**: 구조화 (Markdown 활용)
6. **Markdown 기초**: 주요 문법 요약
7. **Markdown 예시**: Before/After 비교
8. **VS Code 시연 안내**: 라이브 데모 준비
9. **프롬프트 작성 원칙 3**: 예시 기반 학습 (Few-shot)
10. **Few-shot 예시**: 실제 사례
11. **컨텍스트 관리 개념**: 왜 중요한가?
12. **MCP 소개**: Model Context Protocol이란?
13. **MCP 도구 개요**: 다양한 도구 소개
14. **MCP 도구 상세 1**: byterover, sequential thinking

## 5. 인터랙션 설계

- **화면 공유**: 큰 글씨와 명확한 화면 구성
