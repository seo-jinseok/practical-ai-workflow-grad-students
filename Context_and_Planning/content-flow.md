# 강의 콘텐츠 흐름 및 내러티브 설계

## 전체 내러티브 아크

### 스토리라인
"대학원생 여러분의 연구와 학습을 더 효율적이고 창의적으로 만들어줄 생성형 AI 활용법을 배웁니다. 단순히 도구를 사용하는 것을 넘어, 효과적인 소통(Context Engineering)과 체계적인 계획(Planning)을 통해 AI를 진정한 연구 파트너로 만드는 방법을 익힙니다."

### 강의 흐름의 논리적 연결
1. **문제 인식** (도입부): 대학원 연구의 어려움과 AI의 가능성
2. **첫 번째 해결책** (Part 1): AI와 효과적으로 소통하는 법 (Context Engineering)
3. **두 번째 해결책** (Part 2): AI를 활용한 체계적 계획 및 관리 (Planning)
4. **통합 및 실천** (Part 3): 두 가지를 결합한 실전 워크플로우
5. **행동 촉구** (마무리): 오늘부터 시작할 수 있는 구체적 액션

## 섹션별 상세 콘텐츠 흐름

### 도입부 (15분) - "왜 지금 생성형 AI인가?"

#### 오프닝 (3분)
- **Hook**: "여러분은 논문 한 편을 쓰는 데 얼마나 걸리나요?"
- **현실**: 대학원생의 시간 부족, 정보 과부하, 반복 작업의 고통
- **약속**: 이 강의를 통해 연구 생산성을 2배 이상 높일 수 있는 방법 제시

#### 생성형 AI 현황 (7분)
- **현재 상황**: ChatGPT, Claude, Gemini 등 주요 모델 소개
- **통계 자료**: 연구자들의 AI 활용 증가 추세 (최신 데이터)
- **성공 사례**: 해외 유명 대학원생/연구자의 AI 활용 사례
- **한계 인식**: AI가 할 수 있는 것과 할 수 없는 것 명확히 구분
- **윤리적 고려**: 표절, 저작권, 학문적 정직성 문제

#### 청중 연결 (3분)
- **배경 파악**: "AI를 사용해본 적 있는 분?" (손들기)
- **기대사항**: "이 강의에서 무엇을 배우고 싶으신가요?" (2-3명 발표)
- **공감대 형성**: 부산/경남 지역 대학원생의 공통 고민 언급

#### 로드맵 제시 (2분)
- **Part 1 예고**: AI와 대화하는 기술
- **Part 2 예고**: AI로 프로젝트 관리하기
- **Part 3 예고**: 실전 통합 워크플로우
- **시연 안내**: 중간중간 실제 도구 사용 시연

### Part 1: Context Engineering (40분) - "AI와 효과적으로 대화하기"

#### 섹션 도입 (3분)
- **전환**: "AI를 사용해봤지만 원하는 답을 못 받은 경험?"
- **핵심 메시지**: "좋은 질문이 좋은 답을 만든다"
- **학습 목표 제시**: 4가지 핵심 스킬

#### 효과적인 프롬프트 작성 원칙 (15분)

**1. 명확성과 구체성 (4분)**
- **나쁜 예시**: "논문 써줘"
- **좋은 예시**: "교육학 분야 질적 연구 방법론에 관한 석사 논문의 서론 초안을 작성해줘. 연구 주제는 '온라인 학습 환경에서 학습자의 자기조절학습 전략'이고, 3-4페이지 분량으로 작성해줘."
- **원칙**: 5W1H (누가, 무엇을, 언제, 어디서, 왜, 어떻게)
- **실습**: 자신의 연구 주제로 구체적 프롬프트 작성 (1분)

**2. 구조화된 입력 - Markdown 활용 (4분)**
- **Markdown이란**: 간단한 문법으로 구조화된 문서 작성
- **주요 문법**: 제목(#), 목록(-), 강조(**), 코드(```)
- **Before/After 비교**:
  - Before: 긴 문장으로 나열된 요구사항
  - After: Markdown으로 구조화된 명확한 요구사항
- **장점**: AI가 이해하기 쉽고, 재사용 가능하며, 버전 관리 용이

**3. VS Code 시연 (4분)**
- **실제 화면**: VS Code에서 Markdown 파일 작성
- **확장 기능**: Markdown Preview, Markdown All in One
- **실시간 작성**: 논문 개요를 Markdown으로 구조화
- **AI 연동**: 작성한 Markdown을 ChatGPT/Claude에 입력

**4. 예시 기반 학습 (Few-shot Prompting) (3분)**
- **개념**: 예시를 제공하여 AI 학습 유도
- **예시**: 논문 초록 작성 시 좋은 초록 2-3개 제공
- **효과**: 일관된 스타일과 품질 유지

#### 컨텍스트 관리 전략 (12분)

**2. MCP(Model Context Protocol) 소개 (5분)**
- **MCP란**: AI 모델에 외부 컨텍스트를 제공하는 표준 프로토콜
- **왜 필요한가**: AI의 지식 한계 극복, 최신 정보 제공, 개인화된 컨텍스트
- **작동 원리**: 간단한 다이어그램으로 설명

**3. 다양한 MCP 도구 소개 (7분)**
- **byterover**: 코드베이스 이해 및 분석
- **sequential thinking**: 단계적 사고 프로세스 지원
- **context7**: 대화 컨텍스트 관리 및 확장
- **검색 도구들** (brave, duckduckgo, tavily): 실시간 웹 검색
- **task-master-mcp**: 작업 관리 및 추적
- **perplexity**: 학술 검색 및 인용
- **git**: 버전 관리 통합
- **선택 가이드**: 연구 목적에 따른 도구 선택 기준

#### 대학원 연구 사례 적용 (8분)

**1. 논문 작성 지원 (3분)**
- **시나리오**: 문헌 고찰 섹션 작성
- **프로세스**:
  1. 주제 및 키워드 정의 (Markdown)
  2. 검색 MCP로 최신 논문 수집
  3. 구조화된 프롬프트로 요약 요청
  4. 인용 형식 자동 변환
- **결과**: 시간 절약 및 품질 향상

**2. 문헌 조사 및 정리 (3분)**
- **시나리오**: 100편의 논문을 주제별로 분류
- **프로세스**:
  1. 논문 제목과 초록 수집
  2. AI에게 주제 분류 요청 (Few-shot)
  3. 결과를 표 형식으로 정리
  4. 주요 트렌드 분석
- **도구 활용**: Perplexity MCP + Markdown 테이블

**3. 혁신보고서 작성 사례 (2분)**
- **배경**: `idea.md`에 언급된 실제 사례
- **적용**: Context Engineering 기법으로 보고서 품질 향상
- **교훈**: 구조화와 명확한 컨텍스트의 중요성

#### Part 1 마무리 (2분)
- **핵심 요약**: 명확성, 구조화, 컨텍스트 관리
- **실천 과제**: 오늘 배운 기법으로 프롬프트 1개 작성해보기
- **전환**: "이제 이 기법을 프로젝트 관리에 적용해봅시다"

### Part 2: Planning & Task Management (40분) - "AI로 프로젝트 체계적으로 관리하기"

#### 섹션 도입 (3분)
- **전환 질문**: "프로젝트를 시작할 때 가장 어려운 점은?"
- **공통 문제**: 막연함, 우선순위 불명확, 진행 상황 파악 어려움
- **해결책 예고**: Spec-driven planning과 AI 활용

#### Spec-driven Planning 개념 (12분)

**1. 명세 기반 계획의 중요성 (4분)**
- **Spec이란**: 프로젝트의 목표, 범위, 요구사항을 명확히 정의한 문서
- **전통적 방식 vs Spec-driven**:
  - 전통적: 머릿속 계획 → 중간에 방향 상실
  - Spec-driven: 문서화된 계획 → 명확한 방향성
- **장점**: 명확성, 추적 가능성, 협업 용이성, AI 활용 최적화

**2. AI를 활용한 요구사항 분석 (4분)**
- **프로세스**:
  1. 프로젝트 아이디어를 AI에게 설명
  2. AI가 질문을 통해 요구사항 도출
  3. 구조화된 Spec 문서 생성
  4. 검토 및 수정
- **예시**: 석사 논문 프로젝트 Spec 작성
- **템플릿**: 재사용 가능한 Spec 문서 템플릿 제공

**3. GitHub Copilot 시연 (4분)**
- **시나리오**: 연구 계획서 작성
- **시연 내용**:
  - 주석으로 요구사항 작성
  - Copilot이 자동으로 내용 제안
  - Tab으로 수락하며 빠르게 문서 완성
- **강조점**: Spec-driven 접근의 실제 적용

#### AI 기반 작업 분해 및 관리 (15분)

**1. 작업 분해 개념 (3분)**
- **문제**: "논문 쓰기"는 너무 큰 작업
- **해결**: 작은 단위로 분해 (Work Breakdown Structure)
- **예시**:
  - 논문 쓰기 → 서론, 문헌고찰, 방법론, 결과, 결론
  - 서론 → 연구 배경, 연구 문제, 연구 목적, 논문 구성
- **AI 활용**: 자동 작업 분해 및 제안

**2. 작업 분해 방법론 (4분)**
- **프롬프트 예시**: "다음 프로젝트를 2주 안에 완료 가능한 작은 작업들로 분해해줘: [프로젝트 설명]"
- **AI 출력**: 작업 목록, 예상 소요 시간, 의존성
- **실습**: 자신의 프로젝트로 작업 분해 연습 (2분)

**3. 우선순위 설정 및 일정 관리 (4분)**
- **중요도/긴급도 매트릭스**: 아이젠하워 매트릭스 소개
- **AI 활용**: 작업 우선순위 자동 제안
- **타임라인 작성**: Gantt 차트 또는 간단한 일정표
- **도구 연계**: Google Calendar, Notion, Trello 등

**4. GitHub Projects 시연 (4분)**
- **GitHub Projects 소개**: GitHub의 프로젝트 관리 도구
- **시연 내용**:
  - 프로젝트 Spec 작성
  - 자동 작업 분해
  - 이슈 트래킹 연동
- **장점**: 코드와 문서의 통합 관리

#### Review와 Verification 프로세스 (8분)

**1. AI 출력물 검증 전략 (3분)**
- **문제**: AI는 때때로 틀린 정보 제공 (Hallucination)
- **검증 원칙**:
  - 사실 확인: 출처 검증, 교차 확인
  - 논리 확인: 추론 과정 검토
  - 맥락 확인: 연구 목적에 부합하는지
- **도구 활용**: Perplexity로 인용 확인, 검색 MCP로 팩트 체크

**2. 품질 관리 체크리스트 (3분)**
- **체크리스트 항목**:
  - [ ] 연구 질문에 부합하는가?
  - [ ] 논리적 일관성이 있는가?
  - [ ] 인용이 정확한가?
  - [ ] 표절 검사를 통과했는가?
  - [ ] 형식이 요구사항에 맞는가?
- **자동화**: AI로 초기 검토, 사람이 최종 확인

**3. 반복적 개선 프로세스 (2분)**
- **피드백 루프**: 작성 → 검토 → 수정 → 재검토
- **AI 활용**: "다음 내용을 검토하고 개선점을 제안해줘"
- **버전 관리**: Git으로 변경 이력 추적

#### Part 2 마무리 (2분)
- **핵심 요약**: Spec-driven, 작업 분해, 검증
- **실천 과제**: 현재 프로젝트의 Spec 문서 작성해보기
- **전환**: "이제 Part 1과 2를 통합해봅시다"

### Part 3: 실전 워크플로우 통합 (20분) - "모든 것을 하나로"

#### 섹션 도입 (2분)
- **목표**: Context Engineering + Planning을 실제 프로젝트에 적용
- **시나리오 선택**: 논문 작성 또는 데이터 분석 프로젝트

#### End-to-End 워크플로우 시연 (12분)

**시나리오: 석사 논문 작성 프로젝트 (예시)**

**Step 1: Context 설정 및 프롬프트 작성 (3분)**
- **작업**: 연구 주제 정의 및 초기 컨텍스트 구축
- **도구**: VS Code + Markdown
- **내용**:
  ```markdown
  # 연구 프로젝트: 온라인 학습 환경에서의 자기조절학습

  ## 연구 배경
  - 코로나19 이후 온라인 학습 급증
  - 자기조절학습 능력의 중요성 증가

  ## 연구 질문
  1. 온라인 학습 환경에서 학습자들은 어떤 자기조절학습 전략을 사용하는가?
  2. 효과적인 전략과 비효과적인 전략의 차이는 무엇인가?
  ```
- **AI 활용**: 이 컨텍스트를 바탕으로 문헌 조사 방향 제안 받기

**Step 2: 계획 수립 및 작업 분해 (3분)**
- **작업**: Spec 문서 작성 및 작업 분해
- **도구**: GitHub Projects 또는 Notion
- **AI 프롬프트**: "위 연구 프로젝트를 6개월 안에 완료하기 위한 상세 계획을 작성해줘. 각 단계별 작업, 소요 시간, 산출물을 포함해줘."
- **결과**: 주차별 작업 계획, 마일스톤, 체크포인트

**Step 3: 실행 및 진행 관리 (3분)**
- **작업**: 실제 논문 작성 시작
- **도구**: task-master-mcp 시연
- **시연 내용**:
  - 작업 목록 자동 생성
  - 진행 상황 추적
  - 다음 작업 자동 제안
  - 막힌 부분 해결 도움
- **강조**: AI가 프로젝트 매니저 역할

**Step 4: 검증 및 개선 (3분)**
- **작업**: 작성한 내용 검토 및 개선
- **프로세스**:
  1. AI에게 초안 검토 요청
  2. 체크리스트 기반 자가 검토
  3. 동료/지도교수 피드백
  4. AI 도움으로 수정
- **반복**: 만족할 때까지 Step 3-4 반복

#### 다양한 전공별 적용 방안 (4분)

**1. 인문사회계열 (1.5분)**
- **사례**: 질적 연구 데이터 분석
- **적용**:
  - Context: 인터뷰 전사본을 Markdown으로 구조화
  - Planning: 코딩 및 주제 분석 계획
  - 실행: AI로 초기 코딩 제안, 연구자가 검증

**2. 이공계열 (1.5분)**
- **사례**: 실험 데이터 분석 및 논문 작성
- **적용**:
  - Context: 실험 프로토콜 및 데이터를 구조화
  - Planning: 분석 파이프라인 설계
  - 실행: 코드 작성 (GitHub Copilot), 결과 해석 (AI 도움)

**3. 예체능계열 (1분)**
