# Part 2 Review & Verification 체크리스트 모음

이 문서는 다양한 유형의 연구 산출물에 대한 검증 체크리스트를 제공합니다.

## 논문 섹션별 체크리스트

### 1. 서론 (Introduction) 체크리스트

```markdown
# 서론 검증 체크리스트

## ✅ 내용 검증

### 연구 배경
- [ ] 주제의 중요성이 명확히 제시되었는가?
- [ ] 사회적/학술적 맥락이 적절히 설명되었는가?
- [ ] 최신 동향이 반영되었는가? (최근 5년 이내)
- [ ] 논리적 흐름이 자연스러운가?

### 문제 제기
- [ ] 현재의 문제/갭이 명확히 제시되었는가?
- [ ] 선행 연구의 한계가 구체적으로 언급되었는가?
- [ ] 본 연구의 필요성이 설득력 있게 제시되었는가?

### 연구 목적 및 질문
- [ ] 연구 목적이 명확하고 구체적인가?
- [ ] 연구 질문이 측정/답변 가능한가?
- [ ] 연구 목적과 질문이 일치하는가?
- [ ] 연구 질문의 수가 적절한가? (2-4개)

### 연구의 의의
- [ ] 학술적 기여가 명확한가?
- [ ] 실천적 함의가 제시되었는가?
- [ ] 과장되지 않고 현실적인가?

## 📝 형식 검증

### 구조
- [ ] 일반적 → 구체적 순서로 전개되는가?
- [ ] 각 문단이 하나의 주제를 다루는가?
- [ ] 문단 간 연결이 자연스러운가?

### 분량
- [ ] 전체 논문의 10-15% 정도인가?
- [ ] 석사: 8-12페이지, 박사: 15-20페이지 정도인가?

### 인용
- [ ] 주요 주장에 적절한 인용이 있는가?
- [ ] 인용 형식이 일관되는가? (APA, MLA 등)
- [ ] 최신 문헌이 포함되었는가?

## 🤖 AI 검증 프롬프트

```markdown
# 서론 검토 요청

다음은 내 논문의 서론입니다:

[서론 전체 텍스트]

## 검토 요청사항

다음 관점에서 검토하고 개선점을 제안해줘:

1. **논리적 흐름**
   - 문단 간 연결이 자연스러운가?
   - 일반에서 구체로 전개되는가?

2. **명확성**
   - 불명확하거나 모호한 표현
   - 전문 용어 설명 필요 여부

3. **완성도**
   - 누락된 중요 내용
   - 불필요하게 장황한 부분

4. **학술적 수준**
   - 주장에 대한 근거 충분성
   - 인용의 적절성

각 문제점에 대해:
- 해당 부분 인용
- 문제점 설명
- 구체적 개선 방안
```
```

### 2. 방법론 (Methods) 체크리스트

```markdown
# 방법론 검증 체크리스트

## ✅ 내용 검증

### 연구 설계
- [ ] 연구 설계 유형이 명확히 제시되었는가?
- [ ] 설계 선택의 근거가 있는가?
- [ ] 연구 질문에 적합한 설계인가?

### 연구 대상
- [ ] 모집단이 명확히 정의되었는가?
- [ ] 표본 크기가 적절한가? (통계적 검정력)
- [ ] 표집 방법이 구체적으로 기술되었는가?
- [ ] 선정/제외 기준이 명확한가?

### 측정 도구
- [ ] 사용한 도구가 명확히 기술되었는가?
- [ ] 도구의 신뢰도/타당도가 제시되었는가?
- [ ] 도구 선택의 근거가 있는가?
- [ ] 수정한 경우 수정 내용이 기술되었는가?

### 자료 수집 절차
- [ ] 절차가 단계별로 상세히 기술되었는가?
- [ ] 시간, 장소, 방법이 구체적인가?
- [ ] 재현 가능할 정도로 상세한가?
- [ ] 윤리적 고려사항이 언급되었는가?

### 자료 분석
- [ ] 분석 방법이 구체적으로 제시되었는가?
- [ ] 사용한 소프트웨어/도구가 명시되었는가?
- [ ] 분석 절차가 단계별로 기술되었는가?
- [ ] 통계적 유의 수준이 제시되었는가?

## 📝 형식 검증

### 재현 가능성
- [ ] 다른 연구자가 동일하게 수행할 수 있는가?
- [ ] 모든 절차가 구체적으로 기술되었는가?
- [ ] 사용한 도구/재료가 모두 명시되었는가?

### 시제
- [ ] 과거 시제로 일관되게 작성되었는가?
- [ ] 수동태 사용이 적절한가? (분야별 관행)

### 분량
- [ ] 전체 논문의 15-20% 정도인가?
- [ ] 너무 간략하거나 장황하지 않은가?

## 🤖 AI 검증 프롬프트

```markdown
# 방법론 검토 요청

다음은 내 논문의 방법론 섹션입니다:

[방법론 전체 텍스트]

## 검토 요청사항

다음 관점에서 검토해줘:

1. **재현 가능성**
   - 다른 연구자가 동일하게 수행할 수 있는가?
   - 누락된 정보가 있는가?

2. **적절성**
   - 연구 질문에 적합한 방법인가?
   - 더 나은 대안이 있는가?

3. **명확성**
   - 불명확한 절차나 용어
   - 추가 설명이 필요한 부분

4. **윤리적 고려**
   - 연구 윤리 관련 누락 사항
   - IRB 승인 관련 정보

구체적인 개선 방안을 제시해줘.
```
```

### 3. 결과 (Results) 체크리스트

```markdown
# 결과 검증 체크리스트

## ✅ 내용 검증

### 데이터 제시
- [ ] 모든 연구 질문에 대한 결과가 제시되었는가?
- [ ] 결과가 객관적으로 기술되었는가? (해석 없이)
- [ ] 통계 결과가 정확하게 보고되었는가?
- [ ] 유의하지 않은 결과도 보고되었는가?

### 표 (Tables)
- [ ] 각 표에 번호와 제목이 있는가?
- [ ] 표가 자체적으로 이해 가능한가?
- [ ] 표의 형식이 일관되는가?
- [ ] 본문에서 모든 표가 언급되는가?
- [ ] 통계 기호가 정확한가? (*, **, ***)

### 그림 (Figures)
- [ ] 각 그림에 번호와 캡션이 있는가?
- [ ] 그림이 명확하고 읽기 쉬운가?
- [ ] 축 레이블과 범례가 있는가?
- [ ] 본문에서 모든 그림이 언급되는가?
- [ ] 해상도가 충분한가? (300 dpi 이상)

### 통계 보고
- [ ] 기술 통계가 적절히 보고되었는가? (M, SD)
- [ ] 검정 통계량이 정확한가? (t, F, χ², r 등)
- [ ] p-값이 정확히 보고되었는가?
- [ ] 효과 크기가 보고되었는가? (Cohen's d, η² 등)
- [ ] 신뢰구간이 제시되었는가? (필요시)

## 📝 형식 검증

### 구조
- [ ] 연구 질문 순서대로 제시되었는가?
- [ ] 일반적 결과 → 구체적 결과 순인가?
- [ ] 각 서브섹션이 명확히 구분되는가?

### 서술
- [ ] 과거 시제로 작성되었는가?
- [ ] 객관적 톤을 유지하는가?
- [ ] 해석이나 논의가 섞이지 않았는가?

### APA 형식 (예시)
- [ ] 통계 결과: t(df) = value, p = .xxx
- [ ] 상관관계: r(df) = value, p = .xxx
- [ ] ANOVA: F(df1, df2) = value, p = .xxx

## 🤖 AI 검증 프롬프트

```markdown
# 통계 결과 검증 요청

다음은 내 분석 결과입니다:

[통계 출력 결과 붙여넣기]

## 요청사항

1. 분석 방법이 적절한가?
2. 가정 검증이 필요한가?
3. 추가 분석이 필요한가?
4. 결과 해석 시 주의사항은?
5. APA 형식으로 결과 작성
```
```

### 4. 논의 (Discussion) 체크리스트

```markdown
# 논의 검증 체크리스트

## ✅ 내용 검증

### 결과 해석
- [ ] 주요 결과가 모두 해석되었는가?
- [ ] 해석이 결과에 근거하는가? (과도한 추론 없음)
- [ ] 연구 질문에 직접 답하는가?
- [ ] 예상과 다른 결과에 대한 설명이 있는가?

### 선행 연구와의 비교
- [ ] 선행 연구와의 일치/불일치가 논의되었는가?
- [ ] 불일치 시 그 이유가 설명되었는가?
- [ ] 본 연구의 독창적 기여가 명확한가?

### 이론적/실천적 함의
- [ ] 이론적 함의가 제시되었는가?
- [ ] 실천적 함의가 구체적인가?
- [ ] 과장되지 않고 현실적인가?

### 한계점
- [ ] 연구의 한계가 솔직히 제시되었는가?
- [ ] 각 한계점이 구체적인가?
- [ ] 한계점이 결과 해석에 미치는 영향이 논의되었는가?

### 후속 연구 제안
- [ ] 구체적인 후속 연구 방향이 제시되었는가?
- [ ] 본 연구의 한계를 보완하는 방향인가?
- [ ] 실현 가능한 제안인가?

## 📝 형식 검증

### 구조
- [ ] 결과 요약 → 해석 → 함의 → 한계 순인가?
- [ ] 각 서브섹션이 명확히 구분되는가?
- [ ] 논리적 흐름이 자연스러운가?

### 균형
- [ ] 긍정적/부정적 결과가 균형있게 다뤄지는가?
- [ ] 한계점이 과도하게 강조되지 않는가?
- [ ] 함의가 과장되지 않는가?

### 분량
- [ ] 전체 논문의 20-25% 정도인가?
- [ ] 결과 섹션과 비슷하거나 약간 긴가?

## 🤖 AI 검증 프롬프트

```markdown
# 논의 섹션 검토 요청

## 연구 질문
1. [연구 질문 1]
2. [연구 질문 2]

## 주요 결과
- [결과 1]
- [결과 2]
- [결과 3]

## 논의 초안
[논의 섹션 전체 텍스트]

## 요청사항

다음을 검토해줘:

1. **결과 해석의 적절성**
   - 과도한 해석이나 추론이 있는가?
   - 결과에 근거하지 않은 주장이 있는가?

2. **논리적 일관성**
   - 모순되는 주장이 있는가?
   - 논리적 비약이 있는가?

3. **선행 연구와의 연결**
   - 선행 연구와의 비교가 충분한가?
   - 추가로 언급해야 할 연구가 있는가?

4. **한계점**
   - 누락된 중요한 한계점이 있는가?
   - 과도하게 방어적이지 않은가?

개선 방안을 구체적으로 제시해줘.
```
```

### 양적 연구 체크리스트

```markdown
# 양적 연구 품질 체크리스트

## 📊 연구 설계
- [ ] 연구 설계가 연구 질문에 적합한가?
- [ ] 변수가 명확히 정의되고 조작화되었는가?
- [ ] 통제 변수가 적절히 선정되었는가?

## 📏 측정 도구
- [ ] 신뢰도 계수가 보고되었는가? (Cronbach's α ≥ .70)
- [ ] 타당도 증거가 제시되었는가?
- [ ] 도구의 출처가 명확한가?

## 👥 표본
- [ ] 표본 크기가 충분한가? (G*Power 계산)
- [ ] 표집 방법이 적절한가?
- [ ] 응답률이 보고되었는가?
- [ ] 탈락률 및 이유가 보고되었는가?

## 📈 통계 분석
- [ ] 분석 방법이 연구 질문에 적합한가?
- [ ] 가정 검증이 수행되었는가?
- [ ] 다중 비교 시 보정이 적용되었는가?
- [ ] 효과 크기가 보고되었는가?

## 📊 결과 보고
- [ ] 기술 통계가 완전히 보고되었는가?
- [ ] 통계 결과가 APA 형식에 맞는가?
- [ ] 표와 그림이 명확한가?
- [ ] 모든 가설에 대한 결과가 있는가?

3. 추가 분석이 필요한가?

### 질적 연구 체크리스트

```markdown
# 질적 연구 품질 체크리스트

## 🎯 연구 설계
- [ ] 질적 연구 접근법이 명확한가? (현상학, 근거이론 등)
- [ ] 접근법 선택의 근거가 있는가?
- [ ] 연구자의 관점/편견이 언급되었는가?

## 👥 참여자
- [ ] 표집 전략이 적절한가? (목적 표집 등)
- [ ] 참여자 수가 적절한가?
- [ ] 데이터 포화가 달성되었는가?
- [ ] 참여자 특성이 기술되었는가?

## 💬 데이터 수집
- [ ] 인터뷰/관찰 절차가 상세한가?
- [ ] 질문 가이드가 제시되었는가?
- [ ] 녹음/전사 절차가 기술되었는가?
- [ ] 연구자-참여자 관계가 언급되었는가?

## 🔍 데이터 분석
- [ ] 분석 방법이 구체적인가? (주제 분석, 내용 분석 등)
- [ ] 코딩 절차가 단계별로 기술되었는가?
- [ ] 분석의 신뢰도 확보 방법이 있는가?
- [ ] 연구자 간 신뢰도가 보고되었는가? (필요시)

## ✅ 신뢰도 및 타당도
- [ ] 신뢰도 확보 전략이 제시되었는가?
  - 삼각검증 (Triangulation)
  - 구성원 검토 (Member checking)
  - 동료 검토 (Peer debriefing)
- [ ] 타당도 확보 전략이 제시되었는가?
  - 풍부한 기술 (Thick description)
  - 감사 추적 (Audit trail)

## 📝 결과 제시
- [ ] 주제/범주가 명확히 제시되었는가?
- [ ] 각 주제에 대한 설명이 충분한가?
- [ ] 참여자 인용이 적절히 사용되었는가?
- [ ] 인용이 주제를 잘 뒷받침하는가?

## 🤖 AI 활용 검증

```markdown

## 표절 검사 가이드

### 표절 검사 도구
1. **Turnitin** (대학 제공)
   - 가장 널리 사용
   - 학술 데이터베이스 포함
   - 기준: 일반적으로 15% 이하

2. **Copykillers** (한국어)
   - 한국어 논문에 특화
   - RISS 연동

3. **무료 도구**
   - Grammarly (영문)
   - PlagScan (제한적 무료)

### 표절 검사 해석

**유사도 비율 기준**:
- **0-15%**: 일반적으로 허용
- **15-25%**: 주의 필요, 검토 후 수정
- **25% 이상**: 심각, 대폭 수정 필요

**주의사항**:
- 참고문헌 목록은 유사도에서 제외
- 일반적인 방법론 기술은 어느 정도 유사 가능
- 직접 인용은 적절히 표시되었는지 확인

### AI 활용 팁

#### 효과적인 검증 프롬프트 작성

```markdown
# [섹션명] 검증 요청

## 연구 정보
- 연구 질문: [연구 질문]
- 연구 방법: [방법]
- 대상 독자: [독자]

## 검증 대상 텍스트
[검증할 텍스트]

## 요청사항

1. **내용 검증**
   - 연구 질문과의 부합성
   - 논리적 일관성
   - 근거 충분성

2. **형식 검증**
   - 학술적 표현
   - 문법 및 구두점
   - APA 형식 준수

3. **개선 제안**
   - 문제점 3가지
   - 구체적 개선 방안
   - 대안 표현 제시

각 문제점에 대해:
- 위치 (페이지/문단)
- 현재 텍스트 인용
- 문제점 설명
- 개선된 버전 제안
```

#### 검증 결과 활용법

1. **우선순위 분류**
   - Critical: 즉시 수정 필요
   - Major: 다음 수정 때 반영
   - Minor: 선택적 개선

2. **수정 계획 수립**
   - Critical 문제부터 해결
   - 관련 문제 일괄 수정
   - 검토 주기 설정

3. **재검증**
   - 수정 후 AI 재검토 요청
   - 체크리스트 재확인
   - 최종 품질 검증

### 품질 기준

#### 내용 품질
- **충실성**: 연구 질문에 모두 답변
- **정확성**: 사실과 해석 구분
- **논리성**: 주장과 근거의 연결
- **완결성**: 누락된 내용 없음

#### 형식 품질
- **일관성**: 용어, 시제, 형식 통일
- **명확성**: 모호한 표현 없음
- **간결성**: 불필요한 내용 제외
- **정확성**: 문법, 맞춤법 오류 없음

#### 학술적 품질
- **독창성**: 새로운 기여점 포함
- **신뢰성**: 방법론적 엄격성
- **타당성**: 결론의 적절성
- **윤리성**: 연구 윤리 준수
