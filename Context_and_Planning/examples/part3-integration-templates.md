## 사용 가이드

### 템플릿 선택 가이드

**프로젝트 시작 시**:
- 템플릿 1 (통합 프로젝트 마스터 문서) 필수
- 이 문서가 모든 것의 중심

**일일 관리**:
- 템플릿 2 (일일 워크플로우) 선택적
- 복잡한 프로젝트나 습관화 초기에 유용

**주간 관리**:
- 템플릿 3 (주간 통합 리뷰) 강력 권장
- 매주 금요일 30분 투자

**문제 발생 시**:
- 템플릿 5 (문제 해결 프로토콜) 활용
- 체계적 해결로 시간 절약

### 맞춤화 방법

1. **복사**: 필요한 템플릿 복사
2. **수정**: [대괄호] 부분을 자신의 정보로
3. **간소화**: 불필요한 섹션 제거
4. **확장**: 필요한 섹션 추가
5. **저장**: 자신만의 템플릿으로 저장

### 도구 통합

**Notion 사용 시**:
- 템플릿을 Notion 페이지로 변환
- 데이터베이스 기능으로 작업 관리
- AI 대화는 별도 페이지에 기록

**Obsidian 사용 시**:
- Markdown 파일로 그대로 사용
- 링크로 문서 간 연결
- 그래프 뷰로 관계 시각화

**GitHub 사용 시**:
- 저장소에 문서 저장
- Issues로 작업 추적
- Projects로 진행 상황 시각화

**Google Docs 사용 시**:
- Markdown을 일반 문서로 변환
- 댓글 기능으로 검토
- 버전 기록 자동 관리

### 지속 가능한 활용

**습관화 전략**:
1. **작게 시작**: 템플릿 1만 사용
2. **루틴 설정**: 매주 금요일 오후
3. **점진적 확장**: 익숙해지면 다른 템플릿 추가
4. **개선**: 사용하며 자신에게 맞게 수정

**동기 유지**:
- 작은 성과 기록 및 축하
- 절약한 시간 계산
- 품질 향상 사례 기록
- 월간 회고로 성장 확인
