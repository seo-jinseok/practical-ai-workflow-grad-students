## 🎯 AI 시대의 컨텍스트 작성 원칙

### 핵심 개념
```
🧠 AI 컨텍스트 = Background + Specification + Plan + Task
                 (배경)   (명세)    (계획)   (작업)
```

### AI 친화적 컨텍스트 작성 가이드
- **구체적**: 추상적인 표현보다 구체적인 사실과 숫자
- **맥락적**: 연구의 전후 상황과 제약사항 명시
- **업데이트 가능**: 연구 진행 상황 반영 쉬운 구조
- **토큰 효율**: 불필요한 서식 없이 핵심 정보 중심
- **AI 친화적**: Markdown 형식으로 구조화

## 📌 AI 활용 가이드 (이 컨텍스트 사용법)

### GitHub Copilot과 함께 사용하기

#### 1단계: 컨텍스트 복사
- **Copilot Chat 열기**: `Ctrl+Shift+I` (Windows) / `Cmd+Shift+I` (Mac)
- **컨텍스트 제공**: 위 Background + Specification 섹션을 복사하여 붙여넣기
- **질문 추가**: 구체적인 질문을 말미에 추가

#### 2단계: 효과적인 질문 예시
```markdown
**내 연구 상황**:
[위 Background + Specification 섹션 내용]

**구체적인 질문**:
1. ["문헌 조사에서 '온라인 학습 참여도' 관련 핵심 논문 5편 추천해줘"]
2. ["설문지 문항 10개 작성해줘. 행동적, 인지적, 정서적 참여를 각기 포함해줘"]
3. ["Python pandas로 t-test 수행하는 코드 작성해줘"]
```

#### 3단계: 모델 선택 팁
- **긴 문헌 요약**: Claude Sonnet 4.5 (200K context)
- **코드 작성**: GPT-5-Codex 또는 Grok Code Fast 1
- **논문 작성**: Claude Opus 4.1 또는 GPT-5
- **빠른 질문**: GPT-5 mini 또는 Claude Haiku 4.5

#### 4단계: 응답 품질 향상
- **피드백 제공**: "이 답변에서 XXX 부분이 더 구체적이면 좋겠어"
- **맥락 추가**: "우리 연구에서는 [구체적 상황]이 달라서..."
- **반복 개선**: "좋아, 이제 이 결과를 [구체적 목적]에 맞춰서..."

### 2025년 AI 신기능 활용

#### Agent mode 사용법
```
사용자: "Agent mode로 내 연구를 도와줘. 현재 [현재 상황]이야. 
다음 단계로 뭐부터 해야 할까?"

→ AI가 연구 진행 상황 분석 후 구체적 다음 단계 제안
```

#### Custom Instructions 활용
- **AGENTS.md 파일**: `00_컨텍스트/AGENTS.md`에 연구 스타일과 선호도 설정
- **Persistent 컨텍스트**: AI가 이전 대화 기억하며 일관된 지원
- **연구별 맞춤화**: 각 연구 프로젝트마다 다른 지침 적용

```markdown
## 🎯 Background

### 연구자 정보
- **이름**: 김교육
- **소속**: 서울대학교 교육학과 석사과정 2학기
- **전공**: 교육공학 (Educational Technology)
- **지도교수**: 박테크 교수 (전공: 교육공학, 온라인 학습)

### 연구 주제
- **한 문장 요약**: 대학 온라인 수업에서 학습자 참여를 높이는 게임화 전략의 효과 분석
- **키워드**: 온라인 학습, 참여도, 게임화, 대학생, gamification, student engagement, motivation
- **연구 분야**: 교육학 → 교육공학 → 온라인 학습 설계

### 연구 동기
- **문제의식**: 코로나19 이후 온라인 수업이 증가했으나, 학습자 참여도가 대면 수업 대비 현저히 낮음 (평균 40% 감소, 2024년 교육부 통계)
- **선행 연구 gap**: 게임화 연구는 K-12에 집중, 대학 수준 연구 부족. 특히 한국 맥락 연구 거의 없음
- **기대 기여**: 한국 대학생에게 효과적인 게임화 전략 발굴, 실제 수업에 적용 가능한 가이드라인 제시

### 사용 가능한 자원
- **기간**: 2025-03 ~ 2025-08 (6개월)
- **예산**: 제한적 (설문 인센티브 소액만 가능)
- **장비/도구**: Canvas LMS, Python (pandas, scipy), Google Forms, 설문조사 도구
- **협력자**: 혼자 진행, 담당 교수님 1명만 정기적 피드백

### 제약 조건
- 실제 수업 접근 어려움 (학기 중 실험 제한)
- 통제 집단 설정 어려움 (윤리적 이슈)
- 통계 분석 경험 부족 (직접 학습 필요)
```

### 공학 연구 예시 (저조도 객체 검출)
```markdown
## 🎯 Background

### 연구자 정보
- **이름**: 이공학
- **소속**: KAIST 전기전자공학부 박사과정 5학기
- **전공**: 머신러닝, 컴퓨터비전
- **지도교수**: 최AI 교수 (전공: Deep Learning)

### 연구 주제
- **한 문장 요약**: 저조도 환경에서 객체 검출 성능을 개선하는 경량 딥러닝 모델 개발
- **키워드**: object detection, low-light, lightweight model, edge computing, YOLO, real-time processing
- **연구 분야**: 공학 → 전기전자공학 → 컴퓨터비전 → 실시간 객체 검출

### 연구 동기
- **문제의식**: 야간 CCTV, 자율주행 야간 주행에서 객체 검출 정확도 급격히 하락 (정상 대비 40-60% 감소)
- **선행 연구 gap**: 기존 방법은 전처리(image enhancement) 후 검출 → 속도 느림. End-to-end 경량 모델 필요
- **기대 기여**: 실시간 처리 가능한(30 FPS+) 경량 모델, 엣지 디바이스 탑재 가능

### 사용 가능한 자원
- **기간**: 2024-09 ~ 2026-02 (18개월)
- **예산**: 연구비 충분 (GPU 서버, 데이터셋 구매 가능)
- **장비/도구**: 
  - GPU: NVIDIA A100 x4 (학교 서버)
  - 엣지 디바이스: Jetson Nano, Raspberry Pi 4
  - 카메라: 야간 촬영용 IR 카메라 3대
- **협력자**: 공동 연구원 1명, 학부 인턴 2명

### 제약 조건
- 실제 야간 데이터 수집 어려움 (날씨, 시간 제약)
- 라벨링 비용 높음 (외주 필요)
- 엣지 디바이스 메모리 제한 (모델 크기 50MB 이하)
```
