# 문제 해결 프로토콜
## AI를 활용한 연구 문제 해결 가이드

**목적**: 연구 진행 중 발생하는 문제들의 체계적 해결과 예방  
**대상**: 모든 전공의 대학원생  
**효과**: 문제 해결 능력 향상, 지연 예방, 스트레스 감소

---

## 🎯 문제 해결의 중요성

### 왜 체계적 해결이 필요한가?
- **자연스러운 발생**: 연구는 문제로 가득 차 있음
- **일시적 해결의 한계**: 임시방편은 반복적 문제 야기
- **도구적 접근**: AI 도구로 문제를 유형화하여 해결
- **예방적 관리**: 문제 발생 전 사전 방지

### 문제 해결의 핵심 원칙
- **문제를 명료화**: "무엇"이 "언제" "어디서" 발생했는가
- **근본 원인 분석**: 증상 치료가 아닌 원인 해결
- **실행 가능한 해결**: 실행 가능하고 측정 가능한 방안
- **지속적 개선**: 해결 과정을 통해 시스템 개선

---

## 🔍 5단계 문제 해결 프레임워크

### Step 1: 문제 정의 (Problem Definition)

**🔎 문제 명료화**:
```markdown
# 문제 정의 체크리스트

## 기본 정보
- **문제 발생일**: [YYYY-MM-DD]
- **위치**: [연구 단계/상황]
- **긴급도**: [긴급/중요/보통/낮음]
- **영향도**: [전체 프로젝트/일부/작은 영향]

## 문제 기술
- **무엇이 문제인가?**: [구체적 설명]
- **언제 발생하는가?**: [발생 시점/조건]
- **어떤 영향을 주는가?**: [영향 사항]
- **현재 상태는怎样的?**: [현황 설명]

## 문제 예시
좋지 않은 정의: "문헌 조사가 잘 안된다"
좋은 정의: "Elicit에서 'self-regulated learning' 검색 시 검색 결과가 1,000편으로 너무 많고, 2023년 이전 논문이 많이 포함되어 핵심 논문 선별이 어렵다"
```

**AI를 활용한 문제 명료화**:
```markdown
# AI 문제 분석 요청

## 상황 설명
연구를 진행하다가 다음과 같은 상황에 직면했습니다:

"IRB 신청서가 3번이나 반려되었고, 지도교수 피드백을 반영해도 추가 수정이 요구됩니다. 논리적 구조, 방법론 설명, 윤리적 고려에서 계속 문제점을 지적받습니다."

## 분석 요청
위 상황을 바탕으로 다음을 분석해 주세요:

1. **문제의 핵심**: 본질적인 문제는 무엇인가?
2. **근본 원인**: 왜 이런 문제가 발생하는가?
3. **영향도**: 이 문제가 전체 연구에 미치는 영향은?
4. **긴급도**: 언제까지 해결해야 하는가?
5. **해결 우선순위**: 어떤 순서로 해결해야 하는가?

## 맥락 정보
- 연구 단계: IRB 신청
- 남은 시간: 2주
- 지도교수 피드백: 3회 반려
- 동료 지원: 있음
```

### Step 2: AI 분석 (AI Analysis)

```markdown
# AI 근본 원인 분석 요청

## 문제 상황
IRB 신청서가 3번 반려되었으며, 다음과 같은 지적을 받았습니다:
- 논리적 구조: 연구 목적과 방법의 일관성 부족
- 방법론 설명: 샘플 선정 기준이 모호함  
- 윤리적 고려: 참여자 보호 방안이 부족함

## AI 분석 요청
1. **패턴 분석**: 3번 반려된 신청서의 공통问题是?
2. **근본 원인**: 표면적 문제가 아닌 설계 단계의 근본적 문제는?
3. **해결 전략**: IRB 신청서의 핵심 요구사항은?
4. **예방 방안**: 향후 동일 문제를 방지하는 방법은?
5. **도구 활용**: 어떤 AI 도구가 IRB 준비에 가장 도움이 되는가?

## 성공 기준
- 다음 IRB 신청이 1회 승인
- 지도교수 만족도 90% 이상
- 관련 지적사항 사전 해결
```

```markdown
# 근본 원인 분석 결과

## 📊 패턴 분석
**공통 문제점**:
1. **설계 단계 문제**: 연구 설계가 연구 질문과 직접적 연결 부족
2. **구체성 부족**: 추상적 표현过多, 구체적 설명 부족
3. **표준 템플릿 불충분**: IRB 표준 양식에 대한 이해 부족

## 🔍 근본 원인
**핵심 문제**: Part 1의 Spec-driven Planning이 IRB 신청서에 반영되지 않음
- 연구목적과 방법의 간접적 연결
- 실패_FAILURE_YEAH 없는 정량적 목표 설정
- 참여자 보호 방안의 구체적 근거 부족

## 🎯 해결 전략
### 1. Spec 문서 반영 (우선순위: 높음)
- **IRB 신청서 상단**: Spec 요약 1페이지 추가
- **연구 목적**: Part 1의 Background 정리
- **연구 설계**: Spec의 "Research Design" 부분 활용

### 2. IRB 표준 양식 분석 (우선순위: 높음)
- **기존 승인된 IRB**: 5개 분석하여 공통점 도출
- **표준 문구 활용**: 승인가능한 표현방식 채택
- **체크리스트**: 필수 항목 20개 목록화

### 3. 윤리적 고려 강화 (우선순위: 중간)
- **기존 연구 참고**: Similar 연구의 IRB 방법론
- **기관 가이드**: 대학 IRB 안내문에서 고유사항 추출
- **전문가 검토**: 사회과학 IRB 전문교수에게 1회 자문

## 🛡️ 예방 방안
1. **IRB 준비 프로세스**: 연구 시작 시 IRB Checklist 포함
2. **동료 검토**: IRB 제출 전 동료 2명 동시 검토 의무화
3. **AI 활용**: Elicit으로 "IRB approved + 연구유형" 검색하여approved examples 수집

## 🤖 추천 AI 도구
1. **Elicit**: "IRB approved + 연구유형" 검색으로 승인된 예시 찾기
2. **Perplexity**: IRB 요구사항 및 최신 지침 조사
3. **Copilot**: IRB 표준 양식에 따른 구조화된 신청서 작성
4. **Notion**: IRB 체크리스트 및 템플릿 관리

## 📋 실행 계획
**1주차**: Spec 기반 IRB 초안 재작성
**2주차**: 동료 검토 및 수정 (2명)
**3주차**: IRB 전문가 자문 및 최종 제출
**목표**: 1회 승인 (3회 제출 방지)
```

## 🚨 응급 상황 처리 프로토콜

### 긴급 문제 판단 기준

**🔴 최우선 긴급 (24시간 내 해결)**:
```markdown
# 최우선 긴급 상황

## 상황 예시
- **IRB deadline**: 내일 마감인데 신청서 미완성
- **konference abstract**: 익일 마감 연구 제출
- **지도교수 deadline**: 이번 주 금요일 제출 필수
- **final 시험**: 내일 deadline 논문 제출

## 긴급 대응 전략
### 1시간 내 해결 가능한 문제
1. **タスク 우선순위 재설정**
   - 긴급 task만 집중
   - 중요 task는 1-2일 delay
   -緊急 task에 모든 시간 투자

2. **AI 도구 최대 활용**
   - Copilot로 초안 신속 생성
   - Perplexity로 최신 정보 확보
   - Elicit로 필요한 자료 신속 수집

3. **외부 지원 요청**
   - 동료에게 즉시 support 요청
   - 지도교수에게 긴급相談
   - 전문가에게 급격한 자문

### 24시간 내 해결 계획
**0-6시간**:緊急 task 집중 해결
**6-12시간**:동료 검토 및 수정
**12-18시간**:지도교수 최종 검토
**18-24시간**:최종 제출 및 확인

### 성공 지표
- deadline 무사히 준수
- 최소한의 품질 유지 (60% 이상)
- 스트레스 수준 관리 (<4.0/5.0)
```

**🟡 중위험 (1주 내 해결)**:
```markdown
# 중위험 상황

## 상황 예시
- **마일스톤 지연**: 예정된 마일스톤 1주 지연
- **AI 도구 오류**: 주요 AI 도구 사용 불가
- **동료 지원 부족**: 예상보다 지원 받지 못함

## 대응 전략
### 3일 내 해결
1. **문제 분석**: 근본 원인 파악
2. **AI 분석**: 해결 방안 연구
3. **해결 실행**: 단계적 해결 시작

### 1주 내 완료
- 해결 방안 완전 적용
- 성과 측정 및 검증
- 향후 유사 문제 예방 방안 수립

### 성공 지표
- 원래 일정 복구
- 문제 발생 원인과 해결 경험 학습
- 시스템 개선 방안 적용
```

### 자원을 절약하는 문제 해결

**시간 절약형 해결**:
```markdown
# 시간 절약형 해결 전략

## 상황
전문가 자문을 받을 시간과 예산이 부족할 때

## 해결 방안
### AI 중심 해결
1. **AI 도구 최대 활용**
   - Elicit + Perplexity + Copilot 조합
   - 3개 도구로 교차 검증
   - AI 전문가 수준의 information 수집

2. **온라인 리소스 활용**
   - YouTube tutorial 시청
   - 온라인 course 수강
   - University library resource 활용

3. **동료 네트워크 활용**
   - 비공식 동료 자문
   - 관련 연구 group 가입
   - social media 전문가 advice

### 효과적 AI 활용법
**프롬프트 최적화**:
- 구체적 맥락 제공
- 단계별 질문 분리
- 필요한 자료 형식 명시

**多重 확인**:
- 여러 AI 도구로 동일 질문
- AI 결과 교차 검증
- manual sampling 검증

### 시간 절약 체크리스트
- [ ] AI 도구 3개 이상 활용
- [ ] online resource 적극 활용
- [ ] 동료 network 지원 요청
- [ ] 단계적 해결而非 전체적 해결
- [ ] 우선순위 명확화
```

**비용 절약형 해결**:
```markdown
# 비용 절약형 해결 전략

## 상황
전문가 자문이나付费 AI 도구를 사용할 수 없을 때

## 해결 방안
### 무료 도구 집중 활용
1. **무료 AI 도구**
   - Semantic Scholar (무료)
   - ResearchRabbit (무료)
   - GitHub Copilot (무료)

2. **대학 지원 서비스**
   - University library consultation
   - Academic writing center
   - Statistics consulting

3. **시회 네트워크**
   - Professor office hours 활용
   - 동료 연구자 group
   - Alumni network 활용

### 비용 효율성 극대화
**성과 대비 비용 분석**:
- 목적: IRB guidance (무료)
- 해결: University library consultation
- 비용: ₩0, 시간: 2시간
- 효과: 高

**階段적 투자 전략**:
1. 무료 방법 먼저 시도
2. 부족한 부분만 전문가 자문
3. 핵심 부분에만비용 투자
4. 향후 유사 문제 예방 시스템 구축

### 성공 지표
- 무가/저비용 해결책 활용률 80% 이상
- 필수付费 지원 20% 이하
- 장기적 지식 축적 및 역량 강화
- 유사 문제 해결 속도 향상
```

## 📋 문제 해결 체크리스트

### 사전 준비 (Prevention)

**연구 시작 시 체크리스트**:
```markdown
# 연구 시작 전 문제 예방 체크리스트

## 1. AI 도구 준비
- [ ] AI 도구 계정 생성 및 설정 완료
- [ ] 맥락 Engineering 템플릿 준비
- [ ] AI 도구 활용법 연구 완료
- [ ] 도구별 강점/약점 파악

## 2. 사람 지원 채널 준비
- [ ] 지도교수 정례 미팅 일정 설정
- [ ] 동료 연구자 지원 group 가입
- [ ] 전문가 자문 채널 확보 (연락처, 기간, 비용)
- [ ] 동료 검토자 pool 구축 (2-3명)

## 3. 연구 계획 및 구조
- [ ] Spec 문서 상세 작성 완료
- [ ] 5단계 문제 해결 프로토콜 숙지
- [ ] problem solving temporary plan 준비
- [ ] 위험 요인 사전 식별 및 대안 마련

## 4. 리소스 및 예산
- [ ] AI 도구 비용 계획 (무료/학생/유료)
- [ ] 전문가 자문 예산 확보 (필요시)
- [ ] 대안 도구 준비 (backup plan)
- [ ] 시간 버퍼 설정 (예상 시간 20% 추가)
```

### 문제 발생 시 대응 체크리스트

**문제 발생 시 1시간 내**:
```markdown
# 문제 발생 시 1시간 대응 체크리스트

## Step 1: 문제 정지 및 상황 파악
- [ ] 문제 발생 즉시 작업 중단
- [ ] 문제 상황 객관적 기록 (시간, 장소, 영향)
- [ ] 긴급도 및 영향도 평가
- [ ] 기본 정보 수집 (가용 리소스, 제약사항)

## Step 2: 긴급 대응
- [ ] 최우선 해결 사항 판단
- [ ] 즉각 대안 마련 (우회 경로)
- [ ] 필요한 지원 요청 (동료, 지도교수)
- [ ] AI 도구 즉시 활용 상황 판단

## Step 3: 체계적 해결 준비
- [ ] 5단계 문제 해결 프로토콜 시작
- [ ] Step 1 (문제 정의) 실행
- [ ] AI 분석 및 해결방안 찾기
- [ ] 해결 실행 일정 및 방법 확정

## Step 4: 커뮤니케이션
- [ ] 관련 사람들 에게 상황 알림
- [ ] 지도교수에게 문제 상황 보고
- [ ] 동료에게 지원 요청 (필요시)
- [ ] 일정 조정 사항 공지
```

### 해결 후 회고 체크리스트

```markdown
# 문제 해결 완료 후 회고 체크리스트

## 결과 평가

- [ ] 시간 관리 (예상 대비 실제)
- [ ] 품질 평가 (결과물 품질)
- [ ] 스트레스 관리 (수준 및 영향)

## 과정 학습
- [ ] 문제 발생 원인 분석
- [ ] 해결 과정에서 배운 점
- [ ] 효과적이었던 방법론
- [ ] 개선이 필요한 영역

## 시스템 개선
- [ ] 예방 방안 (향후 유사 문제 방지)
- [ ] 대응 방안 (更快한 해결)
- [ ] 리소스 방안 (더 효율적 활용)
- [ ] 네트워크 방안 (更好的支持)

## 일반화
- [ ] 유사한 문제 유형에 대한 일반적 해법
- [ ] 도구별 활용 최적화
- [ ] 시간 투자와 효과의 관계 분석
- [ ] 다른 연구자에게 공유할 인사이트
```
